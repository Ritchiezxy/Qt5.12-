详细信息

所述QDesignerMemberSheetExtension类可以操纵配置使用Qt 设计师的模式用于编辑信号和槽连接时，其上显示窗口组件的成员函数。

QDesignerMemberSheetExtension是一组函数，通常用于查询组件的成员函数，以及在Qt 设计师的信号和槽编辑模式下操作成员函数的外观。例如：


  QDesignerMemberSheetExtension *memberSheet  = 0;
  QExtensionManager manager = formEditor->extensionManager();

  memberSheet = qt_extension<QDesignerMemberSheetExtension*>(manager, widget);
  int index = memberSheet->indexOf(setEchoMode);
  memberSheet->setVisible(index, false);

  delete memberSheet;

在实现自定义组件插件时，QDesignerCustomWidgetInterface::initialize () 函数的参数提供指向Qt 设计师当前QDesignerFormEditorInterface对象（formEditor在上面的示例中）的指针。

可以通过使用 qt_extension() 函数查询Qt 设计师的扩展管理器来检索成员表（和任何其他扩展）。当你想释放扩展时，你只需要删除指针。

所有组件都有一个默认成员表，用于Qt 设计师的信号和槽编辑模式以及组件的成员函数。但是QDesignerMemberSheetExtension还提供了一个用于创建自定义成员表扩展的接口。

警告： Qt 设计师使用QDesignerMemberSheetExtension来简化信号和槽编辑模式。每当请求两个组件之间的连接时，Qt 设计师将查询组件的成员表扩展名。如果组件具有已实现的成员表扩展，则此扩展将覆盖默认成员表。

要创建成员表扩展，您的扩展类必须从QObject和QDesignerMemberSheetExtension继承。然后，由于我们正在实现一个接口，我们必须确保使用Q_INTERFACES () 宏让元对象系统知道它：


  class MyMemberSheetExtension : public QObject,
          public QDesignerMemberSheetExtension
  {
      Q_OBJECT
      Q_INTERFACES(QDesignerMemberSheetExtension)

  public:
      ...
  }

这使得Qt 设计师能够使用qobject_cast () 来查询支持的接口，只使用QObject指针。

在Qt 设计师 中，除非需要扩展，否则不会创建扩展。因此，在实现成员表扩展时，您还必须创建一个QExtensionFactory，即能够创建扩展实例的类，并使用Qt 设计师的扩展管理器注册它。

当需要组件的成员表扩展时，Qt 设计师的扩展管理器将运行其所有注册工厂，为每个调用QExtensionFactory::createExtension () 直到找到第一个能够为该组件创建成员表扩展的工厂. 然后这个工厂将创建扩展的一个实例。如果没有找到这样的工厂，Qt 设计师将使用默认的成员表。

Qt 设计师 中有四种可用的扩展类型：QDesignerContainerExtension、QDesignerMemberSheetExtension、QDesignerPropertySheetExtension和QDesignerTaskMenuExtension。无论请求的扩展与多页容器、成员表、属性表还是任务菜单相关联，Qt 设计师的行为都是相同的。

所述QExtensionFactory类提供的标准扩展工厂，并且还可以用作用于定义扩展工厂的接口。您可以创建一个新的QExtensionFactory并重新实现QExtensionFactory::createExtension () 函数。例如：


  QObject *ANewExtensionFactory::createExtension(QObject *object,
          const QString &iid, QObject *parent) const
  {
      if (iid != Q_TYPEID(QDesignerMemberSheetExtension))
          return 0;

      if (MyCustomWidget *widget = qobject_cast<MyCustomWidget*>
             (object))
          return new MyMemberSheetExtension(widget, parent);

      return 0;
  }

或者您可以使用现有工厂，扩展QExtensionFactory::createExtension () 函数，使工厂也能够创建成员表扩展。例如：


  QObject *AGeneralExtensionFactory::createExtension(QObject *object,
          const QString &iid, QObject *parent) const
  {
      MyCustomWidget *widget = qobject_cast<MyCustomWidget*>(object);

      if (widget && (iid == Q_TYPEID(QDesignerTaskMenuExtension))) {
          return new MyTaskMenuExtension(widget, parent);

      } else if (widget && (iid == Q_TYPEID(QDesignerMemberSheetExtension))) {
          return new MyMemberSheetExtension(widget, parent);

      } else {
          return 0;
      }
  }

有关使用扩展类的完整示例，请参阅任务菜单扩展示例。该示例展示了如何为 Qt 设计师 创建自定义组件插件，以及如何使用QDesignerTaskMenuExtension类将自定义项添加到Qt 设计师的任务菜单。

请参考QExtensionFactory、QExtensionManager和创建自定义组件扩展。
