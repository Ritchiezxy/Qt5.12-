详细信息

QFormBuilder类用于运行时动态构建用户界面从用户界面文件。

QFormBuilder类提供了在运行时动态创建用户界面的机制，该机制基于使用Qt Designer创建的UI文件。例如:


          MyForm::MyForm(QWidget *parent)
              : QWidget(parent)
          {
              QFormBuilder builder;
              QFile file(":/forms/myWidget.ui");
              file.open(QFile::ReadOnly);
              QWidget *myWidget = builder.load(&file, this);
              file.close();

              QVBoxLayout *layout = new QVBoxLayout;
              layout->addWidget(myWidget);
              setLayout(layout);
          }

通过在示例的资源 ( myForm.qrc) 中包含用户界面，我们确保在示例运行时它会出现：


      <!DOCTYPE RCC><RCC version="1.0">
      <qresource prefix="/forms">
         <file>mywidget.ui</file>
      </qresource>
      </RCC>

QFormBuilder扩展了QAbstractFormBuilder基类，其中包含许多用于支持自定义组件插件的函数：

  • pluginPaths () 返回表单构建器在加载自定义组件插件时搜索的路径列表。
  • addPluginPath () 允许使用表单构建器注册其他路径。
  • setPluginPath () 用于用从其他来源获得的列表替换现有的路径列表。
  • clearPluginPaths () 删除所有在表单构建器中注册的路径。
  • customWidgets () 返回插件接口列表，可用于创建已注册自定义组件的新实例。

该QFormBuilder类通常被用于自定义嵌入到Qt设计师中的组件和应用程序。需要在运行时动态生成用户界面的独立应用程序使用QtUiTools模块中的QUiLoader类。

请参考 QAbstractFormBuilder 和 Qt UI Tools。
