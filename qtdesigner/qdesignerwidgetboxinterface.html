详细信息

该QDesignerWidgetBoxInterface类允许您控制Qt设计师的组件框中的目录。

QDesignerWidgetBoxInterface包含一组函数，这些函数通常用于操作Qt 设计师的组件框的目录。

Qt 设计师使用一个 XML 文件来填充它的组件框。该文件的名称是组件框的属性之一，您可以使用fileName () 函数检索它。

QDesignerWidgetBoxInterface还提供了save () 函数，该函数将组件框的目录保存在组件框的文件名属性指定的文件中。如果您对组件框进行了更改，例如将组件放入组件框中，而无需调用save () 函数，则只需调用load () 函数即可恢复原始内容：


          QDesignerWidgetBoxInterface *widgetBox = 0:
          widgetBox = formEditor->widgetBox();

          widgetBox->load();

该QDesignerWidgetBoxInterface类不适合直接实例化。您可以使用QDesignerFormEditorInterface::widgetBox () 函数检索 Qt 设计师 组件框的接口。QDesignerCustomWidgetInterface::initialize () 函数的参数提供了指向Qt 设计师当前QDesignerFormEditorInterface对象（formEditor在上面的示例中）的指针。实现自定义组件插件时，您必须继承 QDesignerCustomWidgetInterface以将您的插件公开给Qt 设计师。

如果您想保存您的更改，同时保留原始目录，您可以使用save () 函数结合setFileName () 函数将您的更改保存到另一个文件中。记得先存储原文件名：


          QString originalFile = widgetBox->fileName();

          widgetBox->setFileName("myWidgetBox.xml");
          widgetBox->save();

然后你可以通过将文件名重置为原始文件并调用load ()来恢复组件框的原始目录：


          widgetBox->setFileName(originalFile);
          widgetBox->load();

以类似的方式，您以后可以使用自定义的 XML 文件：


          if (widgetBox->filename() != "myWidgetBox.xml") {
              widgetBox->setFileName("myWidgetBox.xml");
              widgetBox->load();
          }

请参考 QDesignerFormEditorInterface。

[pure virtual] QString QDesignerWidgetBoxInterface::fileName() const

返回Qt 设计师当前用于填充其组件框的 XML 文件的名称。

请参考 setFileName().

[pure virtual] bool QDesignerWidgetBoxInterface::load()

通过加载（或重新加载）当前指定的 XML 文件来填充Qt 设计师的组件框。如果文件加载成功则返回真；否则为假。

请参考 setFileName().

[pure virtual] bool QDesignerWidgetBoxInterface::save()

将Qt 设计师的组件框的目录保存在fileName () 函数指定的文件中。如果内容保存成功则返回true；否则为假。

请参考fileName ()和setFileName ()。

[pure virtual] void QDesignerWidgetBoxInterface::setFileName(const QString &fileName)

将Qt 设计师将用于填充其组件框的 XML 文件设置为fileName。您必须调用load () 以使用新的 XML 文件更新组件框。

请参考fileName ()和load ()。
