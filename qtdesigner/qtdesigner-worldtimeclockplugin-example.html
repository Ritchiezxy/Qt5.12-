世界时间时钟插件示例

为使用信号和槽的Qt 设计师创建自定义组件插件。

在这个例子中，我们通过引入信号和槽的概念，简单地扩展了自定义控件插件示例及其自定义控件（基于模拟时钟示例）。

世界时间时钟插件示例包含两个类：

  • WorldTimeClock 是一个带有时针和分针的自定义时钟组件，每隔几秒钟就会自动更新一次。
  • WorldTimeClockPlugin将WorldTimeClock类公开给Qt 设计师。

首先，我们将通过提供信号和插槽来查看WorldTimeClock扩展自定义组件插件示例AnalogClock类的类。然后我们将快速查看WorldTimeClockPlugin该类，但该类在大多数情况下与自定义组件插件示例的实现相同。

最后我们看一下插件的项目文件。自定义组件插件的项目文件需要一些附加信息以确保它们在Qt 设计师 中工作。这也包含在自定义组件插件示例中，但由于其重要性（自定义组件插件依赖于Qt 设计师提供的组件，必须在我们使用的项目文件中指定）我们将在此处重复。

WorldTimeClock 类

在WorldTimeClock类继承QWidget的，并且是一个自定义时钟组件与时针和分针是每隔几秒钟自动更新。这个例子与自定义组件插件示例的不同之处在于，在自定义组件类中引入了信号和槽：


  class QDESIGNER_WIDGET_EXPORT WorldTimeClock : public QWidget
  {
      Q_OBJECT

  public:
      explicit WorldTimeClock(QWidget *parent = nullptr);

  public slots:
      void setTimeZone(int hourOffset);

  signals:
      void updated(QTime currentTime);

  protected:
      void paintEvent(QPaintEvent *event) override;

  private:
      int timeZoneOffset = 0;
  };

请注意QDESIGNER_WIDGET_EXPORT宏的使用。这是确保Qt 设计师可以在某些平台上创建组件实例所必需的，但最好在所有平台上使用它。

我们setTimeZone()使用关联timeZoneOffset变量声明插槽，并声明一个updated()信号，该信号将当前时间作为参数，并在重新绘制组件时发出。

例如，在Qt 设计师的工作区中，我们可以使用Qt 设计师编辑信号和槽的模式将WorldTimeClock组件的updated()信号连接到QTimeEdit的setTime()槽。

我们还可以将QSpinBox的valueChanged()信号连接到WorldTimeClock的setTimeZone()槽。

WorldTimeClockPlugin 类

WorldTimeClockPlugin 类向Qt设计师公开了WorldTimeClock 类。它的定义等价于自定义组件插件示例中的插件类。特定于这个自定义组件的类定义的唯一部分是类名:


  class WorldTimeClockPlugin : public QObject,
                               public QDesignerCustomWidgetInterface
  {
      Q_OBJECT
      Q_PLUGIN_METADATA(IID "org.qt-project.Qt.QDesignerCustomWidgetInterface")
      Q_INTERFACES(QDesignerCustomWidgetInterface)

  public:
      explicit WorldTimeClockPlugin(QObject *parent = nullptr);

      bool isContainer() const override;
      bool isInitialized() const override;
      QIcon icon() const override;
      QString domXml() const override;
      QString group() const override;
      QString includeFile() const override;
      QString name() const override;
      QString toolTip() const override;
      QString whatsThis() const override;
      QWidget *createWidget(QWidget *parent) override;
      void initialize(QDesignerFormEditorInterface *core) override;

  private:
      bool initialized = false;
  };

插件类为Qt 设计师提供了关于我们插件的基本信息，例如它的类名和它的包含文件。此外，它知道如何创建WorldTimeClockPlugin组件的实例。WorldTimeClockPlugin还定义了initialize()函数，该函数在插件加载到Qt 设计师后调用。该函数的QDesignerFormEditorInterface参数为插件提供了通往所有Qt 设计师 API 的网关。

在WorldTimeClockPlugin类继承两者的QObject和QDesignerCustomWidgetInterface。重要的是要记住，在使用多重继承时，要确保使用Q_INTERFACES () 宏使元对象系统知道所有接口（即不继承Q_OBJECT的类）。这使得Qt 设计师能够使用qobject_cast (QObject * object) 来查询支持的接口，只使用QObject指针。

实现WorldTimeClockPlugin也等价于自定义组件插件示例中的插件接口实现（只是类名和QDesignerCustomWidgetInterface::domXml ()的实现不同）。要记住的主要事情是使用Q_PLUGIN_METADATA () 宏导出WorldTimeClockPlugin用于Qt 设计师的类：


      Q_PLUGIN_METADATA(IID "org.qt-project.Qt.QDesignerCustomWidgetInterface")

没有这个宏，Qt 设计师 就无法使用组件。

项目文件：worldtimeclockplugin.pro

自定义组件插件的项目文件需要一些附加信息以确保它们在Qt 设计师 中按预期工作：


  QT          += widgets uiplugin
  CONFIG     += plugin
  TEMPLATE    = lib

EMPLATE 变量的值与CONFIG 变量中的关键字plugin结合在一起，使得qmake 将自定义组件创建为插件库。

该QT变量包含关键字uiplugin。该插件类型通过实现抽象接口QDesignerCustomWidgetInterface或QDesignerCustomWidgetCollectionInterface为自定义组件创建提供工厂函数，适用于QUiLoader。它不依赖于Qt 设计师库。访问Qt 设计师 的其他接口以实现容器扩展或其他Qt 设计师特定功能的插件遵循不同的规则，并由其他示例涵盖。

组件的头文件和源文件以通常的方式声明，此外我们提供了插件接口的实现，以便Qt 设计师可以使用自定义组件。


  HEADERS     = worldtimeclock.h \
                worldtimeclockplugin.h
  SOURCES     = worldtimeclock.cpp \
                worldtimeclockplugin.cpp

确保插件安装在Qt 设计师搜索到的位置很重要。我们通过为项目指定目标路径并将其添加到要安装的项目列表来完成此操作：


  target.path = $$[QT_INSTALL_PLUGINS]/designer
  INSTALLS += target

自定义组件创建为库，并在安装项目时与其他Qt 设计师插件一起安装（使用make install或等效的安装过程）。后面我们会通过Q_PLUGIN_METADATA ()宏导出相关的widget信息，确保它被Qt 设计师识别为插件。

请注意，如果您希望插件出现在 Visual Studio 集成中，则必须在发布模式下构建插件，并且必须将它们的库复制到集成安装路径中的插件目录中（例如，请参阅 参考资料C:/program files/trolltech as/visual studio integration/plugins）。

有关插件的更多信息，请参阅如何创建 Qt 插件文档。
