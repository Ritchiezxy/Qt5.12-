详细信息

QDesignerCustomWidgetInterface 类允许Qt 设计师 访问和构建自定义组件。

QDesignerCustomWidgetInterface提供了一个带有界面的自定义组件。该类包含一组函数，必须对其进行子类化以返回有关组件的基本信息，例如其类名和其头文件的名称。必须实现其他功能以在加载插件时初始化插件，并构造自定义组件的实例供Qt 设计师使用。

在实现自定义组件时，您必须继承 QDesignerCustomWidgetInterface以将您的组件公开给Qt 设计师。例如，这是自定义组件插件示例中使用的插件的声明，它允许Qt 设计师使用模拟时钟自定义组件：


  class AnalogClockPlugin : public QObject, public QDesignerCustomWidgetInterface
  {
      Q_OBJECT
      Q_PLUGIN_METADATA(IID "org.qt-project.Qt.QDesignerCustomWidgetInterface")
      Q_INTERFACES(QDesignerCustomWidgetInterface)
  public:
      explicit AnalogClockPlugin(QObject *parent = nullptr);

      bool isContainer() const override;
      bool isInitialized() const override;
      QIcon icon() const override;
      QString domXml() const override;
      QString group() const override;
      QString includeFile() const override;
      QString name() const override;
      QString toolTip() const override;
      QString whatsThis() const override;
      QWidget *createWidget(QWidget *parent) override;
      void initialize(QDesignerFormEditorInterface *core) override;

  private:
      bool initialized = false;
  };

注意，类定义中唯一特定于这个特定自定义组件的部分是类名。此外，由于我们正在实现一个接口，我们必须确保使用Q_INTERFACES()宏让元对象系统知道它。这使得Qt 设计师能够使用qobject_cast()函数来转换QObject指针来查询所支持的接口。

在Qt 设计师加载自定义组件插件后，它调用接口的initialize()函数来设置它可能需要的任何资源。使用QDesignerFormEditorInterface参数调用该函数，该参数为插件提供了通往所有Qt 设计师 API的网关。

Qt 设计师通过使用合适的父组件调用插件的createWidget () 函数来构造自定义组件的实例。插件必须构造并返回具有指定父组件的自定义组件的实例。

使用Q_PLUGIN_METADATA()宏导出自定义组件插件到Qt 设计师 。例如，如果一个名为libcustomwidgetplugin.so (在Unix上)或libcustomwidget.dll(在Windows上)的库包含一个名为MyCustomWidget的组件类，我们可以通过在包含插件头的文件中添加以下行来导出它:


  Q_PLUGIN_METADATA(IID "org.qt-project.Qt.QDesignerCustomWidgetInterface")

这个宏确保Qt 设计师可以访问和构建自定义组件。没有这个宏，Qt 设计师 就无法使用它。

实现自定义组件插件时，您将其构建为单独的库。如果要在同一个库中包含多个自定义组件插件，则必须另外子类QDesignerCustomWidgetCollectionInterface。

警告：如果您的自定义组件插件包含QVariant属性，请注意仅支持以下类型：

  • QVariant::ByteArray
  • QVariant::Bool
  • QVariant::Color
  • QVariant::Cursor
  • QVariant::Date
  • QVariant::DateTime
  • QVariant::Double
  • QVariant::Int
  • QVariant::Point
  • QVariant::Rect
  • QVariant::Size
  • QVariant::SizePolicy
  • QVariant::String
  • QVariant::Time
  • QVariant::UInt

有关使用QDesignerCustomWidgetInterface类的完整示例，请参阅自定义组件示例。该示例展示了如何为Qt 设计师创建自定义组件插件。

请参阅 QDesignerCustomWidgetCollectionInterface和为 Qt 设计师 创建自定义组件。

初始化组件以使用指定的formEditor 接口。

请参考 isInitialized()。

[pure virtual] bool QDesignerCustomWidgetInterface::isContainer() const

如果打算将自定义组件用作容器，则返回 true；否则返回false。

大多数自定义组件不用于保存其他组件，因此它们的此函数实现将返回 false，但自定义容器将返回 true 以确保它们在Qt 设计师 中正确运行。

[virtual] bool QDesignerCustomWidgetInterface::isInitialized() const

如果组件已初始化，则返回 true；否则返回false。

请参考 initialize()。

[pure virtual] QString QDesignerCustomWidgetInterface::name() const

返回接口提供的自定义组件的类名。

返回的名称必须与用于自定义组件的类名称相同。

[pure virtual] QString QDesignerCustomWidgetInterface::toolTip() const

在工具提示中返回Qt 设计师可以使用的组件的简短描述。

[pure virtual] QString QDesignerCustomWidgetInterface::whatsThis() const

在组件的“What's This?”帮助中返回可由Qt 设计师 使用的组件的描述。

宏文档

QDESIGNER_WIDGET_EXPORT

这个宏用于定义自定义组件，以确保它们正确地从插件导出，用于Qt 设计师。

在某些平台上，Qt 设计师创建新组件所需的符号会被构建系统从插件中删除，使它们无法使用。使用此宏可确保符号保留在这些平台上，并且对其他平台没有副作用。

例如，世界时间时钟插件示例使用以下声明导出自定义组件类：


  class QDESIGNER_WIDGET_EXPORT WorldTimeClock : public QWidget
  {
      Q_OBJECT
      ...
  };

这个函数是在 Qt 4.1 中引入的。
