详细信息

QExtensionFactory类允许您创建一个能够在Qt Designer中创建自定义扩展实例的工厂。更多...

在Qt 设计师 中，除非需要扩展，否则不会创建扩展。因此，在实现自定义扩展时，您还必须创建一个QExtensionFactory，即能够创建扩展实例的类，并使用Qt 设计师的扩展管理器注册它。

QExtensionManager类为Qt设计师提供了扩展管理工具。当需要扩展时，Qt Designer的扩展管理器将遍历所有已注册的工厂，为每个工厂调用QExtensionFactory::createExtension，直到找到第一个能够为选定对象创建请求扩展的工厂。然后，该工厂将创建扩展的实例。

Qt 设计师 中有四种可用的扩展类型：QDesignerContainerExtension、QDesignerMemberSheetExtension、QDesignerPropertySheetExtension和QDesignerTaskMenuExtension。无论请求的扩展与多页容器、成员表、属性表还是任务菜单相关联，Qt 设计师 的行为都是相同的。

您可以创建一个新的QExtensionFactory并重新实现QExtensionFactory::createExtension () 函数。例如：


          QObject *ANewExtensionFactory::createExtension(QObject *object,
                  const QString &iid, QObject *parent) const
          {
              if (iid != Q_TYPEID(QDesignerContainerExtension))
                  return 0;

              if (MyCustomWidget *widget = qobject_cast<MyCustomWidget*>
                     (object))
                  return new MyContainerExtension(widget, parent);

              return 0;
          }

或者您可以使用现有工厂，扩展QExtensionFactory::createExtension () 函数，使工厂也能够创建您的扩展。例如：


          QObject *AGeneralExtensionFactory::createExtension(QObject *object,
                  const QString &iid, QObject *parent) const
          {
              MyCustomWidget *widget = qobject_cast<MyCustomWidget*>(object);

              if (widget && (iid == Q_TYPEID(QDesignerTaskMenuExtension))) {
                  return new MyTaskMenuExtension(widget, parent);

              } else if (widget && (iid == Q_TYPEID(QDesignerContainerExtension))) {
                  return new MyContainerExtension(widget, parent);

              } else {
                  return 0;
              }
          }

有关使用QExtensionFactory类的完整示例，请参阅任务菜单扩展示例。该示例展示了如何为 Qt 设计师 创建自定义组件插件，以及如何使用QDesignerTaskMenuExtension类将自定义项添加到 Qt 设计师 的任务菜单。

参考QExtensionManager和QAbstractExtensionFactory。
