常见的术语

术语定义
QML编写QML应用程序的语言。语言体系结构和引擎由Qt QML模块实现。
Qt QuickQML语言的类型和功能的标准库，由QtQuick模块提供，可以通过 "import QtQuick 2.3"访问。
Type（类型）在QML中，类型 可以引用基本类型或QML对象类型。。

QML语言提供了许多内置的基本类型，基本类型, Qt Quick模块为构建QML应用程序提供了各种Qt Quick 类型 类型还可以由第三方开发人员通过(modules（模块）） )或应用程序开发人员通过QML文档（类似于术语html文档）在应用程序本身中提供。

有关详细信息，请参阅QML类型系统。

Basic Type（基本类型）基本类型 是简单的类型，如int, string 以及 bool。 与object types（对象类型）不同，对象不能从基本类型实例化;例如，不可能使用属性、方法、信号等创建int对象。

基本类型构建在QML语言中，而对象类型不能使用，除非导入了适当的module（模块）。

有关详细信息，请参阅QML类型系统。

Object Type（对象类型）Object Type（QML对象类型） 是可以由QML引擎实例化的类型。

QML类型可以由以大写字母开头的. QML文件中的文档定义，也可以由基于QObject的C++类定义。

有关详细信息，请参阅QML类型系统。

Object（对象）QML对象是QML Object Type（QML对象类型）的实例。

这些对象是由引擎在处理object declarations（对象声明）时创建的，对象声明指定要创建的对象和为每个对象定义的属性。

此外，可以在运行时通过Component.createObject()和Qt.createQmlObject()动态创建对象。

参见Lazy Instantiation（延迟实例化）。

Component（组件）Component（组件）是创建QML对象或对象树的模板。它是在QML引擎加载文档时生成的。一旦加载了它，就可以使用它实例化它所表示的对象或对象树。

此外，Component（组件） 类型是一种特殊类型，可以用于在文档中内联声明组件。还可以通过Qt.createComponent()动态创建组件对象，从而动态创建QML对象。

Document（文档）QML Document（QML文档）是一段自包含的QML源代码，它以一个或多个导入语句开始，包含一个顶级对象声明。文档可以驻留在.qml文件或文本字符串中。

如果将其放在名称以大写字母开头的. QML文件中，引擎将该文件识别为QML类型的定义。顶级对象声明封装了将由类型实例化的对象树。

Property（属性）属性是具有名称和关联值的对象类型的属性; 这个值可以从外部读取(在大多数情况下，也可以写入)。

一个对象可以有一个或多个属性。一些属性与画布相关(例如., x, y, width, height, and opacity（透明度）) 而另一些属性可能是特定于该类型的数据(例如.Text（文本） 的"text" 属性）。

有关详细信息，请参阅QML对象属性。

Binding（绑定）绑定是一个"绑定"到属性的JavaScript表达式。 属性在任意时刻的值将是通过计算该表达式返回的值得到的。

请参阅属性绑定

Signal（信号）信号是来自QML对象的通知。当一个对象发出一个信号时，其他对象可以通过信号处理程序接收和处理这个信号。

QML对象的大多数属性都有一个更改信号，以及一个关联的更改信号处理程序，客户端可以定义该更改信号处理程序来实现功能。例如，可以在应用程序中定义MouseArea 实例的处理程序"onClicked()" 来处理播放声音事件。

有关详细信息，请参见信号和处理程序事件系统。

Signal Handler（信号处理器）信号处理程序是由信号触发的表达式(或函数)。它在C++中也称为"slot"

有关详细信息，请参见信号和处理程序事件系统。

Lazy Instantiation（延迟实例化）可以在运行时"lazily（延迟）"实例化对象实例，以避免在需要之前执行不必要的工作。Qt Quick提供了Loader(加载器)类型，使延迟实例化更加方便。
