Qt Quick 提供的 C++ 扩展点

所有 QML 应用程序都可以从 C++ 扩展，以便使用在 C++ 代码中实现的附加功能或提供基于 C++ 的 QML 插件。集成 QML 和 C++文档中介绍了从 C++ 扩展 QML 的主题。

此外，Qt Quick 模块为 C++ 开发人员提供了几个特定于该模块的扩展和集成点。特别是，它允许 C++ 开发人员创建和注册可由 Qt Quick 呈现的自定义QQuickItem派生类型。它还提供了几个与场景图相关的类，允许开发人员定义自己的渲染原语。

用户定义的 QQuickItem 派生类型

虽然 Qt Quick 模块已经提供了丰富的可视项类型库供 QML 应用程序使用，但一些开发人员可能希望在 C++ 中定义他们自己的项派生类型并将它们公开给 QML 类型系统。最简单的方法是继承 QQuickItem，它是 Qt Quick 模块中所有视觉类型的基本类型。有关更多详细信息，请参阅QQuickItem文档。

场景图相关类

Qt Quick 2 使用基于 OpenGL ES 2.0 或 OpenGL 2.0 的专用场景图进行渲染。为图形使用场景图而不是传统的命令式绘画系统（QPainter和类似系统）意味着要渲染的场景可以保留在帧之间，并且要渲染的完整图元集在渲染开始之前是已知的。这为许多优化打开了大门，例如批处理 OpenGL 绘制调用以最小化状态更改或丢弃被遮挡的图元。

Qt Quick C++ API提供了各种类来支持在C++中创建自定义节点。

有关详细信息，请参阅Qt Quick 场景图文档。

像素图和线程图像支持

虽然 QML 引擎允许 QML 应用程序从文件系统或网络资源加载图像，但某些应用程序可能需要从基于 C++ 的进程加载图像的附加选项。这可以通过QQuickImageProvider类来实现，该类为 QML 应用程序提供对像素图加载和线程图像请求的支持。任何通过特殊的“image:”URL 方案请求图像的 QML 应用程序都将被定向到适当的图像提供者以加载图像。

有关更多信息，请参阅QQuickImageProvider文档。
