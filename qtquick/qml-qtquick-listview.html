详细信息

ListView显示来自内置 QML 类型（如ListModel和XmlListModel ）创建的模型的数据，或在 C++ 中定义的从QAbstractItemModel或QAbstractListModel继承的自定义模型类。

ListView有一个model ，它定义了要显示的数据，还有一个delegate，它定义了数据的显示方式。ListView中的项水平或垂直布局。列表视图本质上是可滑动的，因为ListView继承自Flickable。

示例用法

以下示例显示了在名为 的文件中定义的简单列表模型的定义ContactModel.qml：


  import QtQuick 2.0

  ListModel {
      ListElement {
          name: "Bill Smith"
          number: "555 3264"
      }
      ListElement {
          name: "John Brown"
          number: "555 8426"
      }
      ListElement {
          name: "Sam Wise"
          number: "555 0473"
      }
  }

另一个组件可以在ListView中显示此模型数据，如下所示：


  import QtQuick 2.0

  ListView {
      width: 180; height: 200

      model: ContactModel {}
      delegate: Text {
          text: name + ": " + number
      }
  }

这里，ListViewContactModel为其模型创建一个组件，并为其委托创建一个Text。该视图将为模型中的每个项创建一个新的Text组件。请注意，委托能够直接访问模型name和number数据。

改进的列表视图如下所示。委托在视觉上得到了改进，并被移动到一个单独的contactDelegate组件中。


  Rectangle {
      width: 180; height: 200

      Component {
          id: contactDelegate
          Item {
              width: 180; height: 40
              Column {
                  Text { text: '<b>Name:</b> ' + name }
                  Text { text: '<b>Number:</b> ' + number }
              }
          }
      }

      ListView {
          anchors.fill: parent
          model: ContactModel {}
          delegate: contactDelegate
          highlight: Rectangle { color: "lightsteelblue"; radius: 5 }
          focus: true
      }
  }

当前选中的项使用highlight属性用蓝色Rectangle突出显示，focus 设置为true，以启用列表视图的键盘导航。列表视图本身就是一个焦点范围(更多细节请参阅Qt Quick中的键盘焦点章节)。

托会根据需要进行实例化，并且可能随时被销毁。它们是ListView的contentItem的父级，而不是视图本身。状态永远不应该存储在委托中。

ListView将许多属性附加到委托的根项，例如ListView.isCurrentItem. 在下面的示例中，根委托项可以直接访问此附加属性作为ListView.isCurrentItem，而子contactInfo对象必须将此属性引用为wrapper.ListView.isCurrentItem。


  ListView {
      width: 180; height: 200

      Component {
          id: contactsDelegate
          Rectangle {
              id: wrapper
              width: 180
              height: contactInfo.height
              color: ListView.isCurrentItem ? "black" : "red"
              Text {
                  id: contactInfo
                  text: name + ": " + number
                  color: wrapper.ListView.isCurrentItem ? "red" : "black"
              }
          }
      }

      model: ContactModel {}
      delegate: contactsDelegate
      focus: true
  }

注意：视图不会自动启用剪辑。如果视图没有被其他项或屏幕剪辑，则需要设置clip: true以便很好地剪辑视图外的项。

ListView布局

ListView中项的布局可以由以下属性控制：

  • orientation - 控制项是水平流动还是垂直流动。该值可以是 Qt.Horizo​​ntal 或 Qt.Vertical。
  • layoutDirection - 控制水平方向视图的水平布局方向：即项是否从视图的左侧布局到右侧，反之亦然。该值可以是 Qt.LeftToRight 或 Qt.RightToLeft。
  • verticalLayoutDirection - 控制垂直方向视图的垂直布局方向：即项是否从视图顶部向下朝向视图底部布局，反之亦然。此值可以是ListView .TopToBottom 或ListView .BottomToTop。

默认情况下，ListView具有垂直方向，并且项从上到下排列。下表显示了ListView可以具有的不同布局，具体取决于上面列出的属性的值。

可滑动的方向

默认情况下，垂直ListView将flickableDirection设置为Flickable.Vertical，而水平ListView将其设置为Flickable.Horizo​​ntal。此外，垂直ListView仅计算（估计）contentHeight，水平ListView仅计算contentWidth。另一个维度设置为-1。

从 Qt 5.9 (Qt Quick 2.9) 开始，可以制作一个可以双向滑动的ListView 。为此，可以将flickableDirection设置为Flickable.AutoFlickDirection或Flickable.AutoFlickIfNeeded ，并且必须提供所需的contentWidth或contentHeight 。


  ListView {
      width: 180; height: 200

      contentWidth: 320
      flickableDirection: Flickable.AutoFlickDirection

      model: ContactModel {}
      delegate: Row {
          Text { text: '<b>Name:</b> ' + name; width: 160 }
          Text { text: '<b>Number:</b> ' + number; width: 160 }
      }
  }

属性文档

add : Transition

此属性保存要应用于添加到视图中的项的转换。

例如，下面是一个指定这种转换的视图：


  ListView {
      ...
      add: Transition {
          NumberAnimation { properties: "x,y"; from: 100; duration: 1000 }
      }
  }

每当将一个项添加到上述视图中时，该项将从位置 (100,100) 动画到其在视图中的最终 x,y 位置，超过一秒。过渡只适用于添加到视图中的新项；它不适用于以下因添加新项而被取代的项。要为置换的项设置动画，请设置置换或addDisplaced属性。

有关如何使用视图转换的更多详细信息和示例，请参阅ViewTransition文档。

注意：此过渡不适用于最初填充视图或视图模型更改时创建的项。（在这些情况下，将应用填充过渡。）此外，此过渡不应为新项的高度设置动画；这样做会导致新项下方的任何项被布置在错误的位置。相反，高度可以在委托的 onAdd 处理程序中进行动画处理。

请参考 addDisplaced，populate 和 ViewTransition。


addDisplaced : Transition

此属性保存要应用于视图中的项的转换，这些项因向视图中添加其他项而移位。

例如，下面是一个指定这种转换的视图：


  ListView {
      ...
      addDisplaced: Transition {
          NumberAnimation { properties: "x,y"; duration: 1000 }
      }
  }

每当将项添加到上述视图时，新项下方的所有项都会移位，导致它们在视图中向下移动（或横向移动，如果水平方向）。当发生这种位移时，项在视图中向其新 x,y 位置的移动将由NumberAnimation在一秒钟内按照指定的方式进行动画处理。此过渡不适用于已添加到视图中的新项；要为添加的项设置动画，请设置add属性。

如果一个项同时被多种类型的操作移位，则没有定义是否将应用 addDisplaced、moveDisplaced或removeDisplaced转换。此外，如果不需要根据项是否被添加、移动或删除操作移位来指定不同的转换，请考虑改为设置置换属性。

有关如何使用视图转换的更多详细信息和示例，请参阅ViewTransition文档。

注意：此过渡不适用于最初填充视图或视图模型更改时创建的项。在这些情况下，将改为应用填充转换。

请参考 displaced，add，populate 和 ViewTransition。


cacheBuffer : int

此属性确定代理是否保留在视图的可见区域之外。

如果此值大于零，则视图可能会保留尽可能多的委托实例化，因为它可以容纳在指定的缓冲区中。例如，如果在垂直视图中，代理的高度为 20 像素并cacheBuffer设置为 40，则最多可以创建/保留可见区域上方的 2 个代理和下方的 2 个代理。缓冲的委托是异步创建的，允许跨多个帧进行创建并减少跳帧的可能性。为了提高绘制性能，可见区域之外的代表不会被绘制。

此属性的默认值取决于平台，但通常是大于零的值。负值被忽略。

请注意，cacheBuffer 不是像素缓冲区——它只维护额外的实例化委托。

注意：设置此属性不能替代创建高效的委托。它可以以额外的内存使用为代价来提高滚动行为的平滑度。委托中的对象和绑定越少，滚动视图的速度就越快。重要的是要意识到设置 cacheBuffer 只会推迟由缓慢加载代理引起的问题，它不是这种情况的解决方案。

cacheBuffer 在displayMarginBeginning或displayMarginEnd指定的任何显示边距之外运行。


count : int

此属性保存视图中的项数。


currentIndex : int

该currentIndex属性保存当前项的索引，并currentItem保存当前项。将 currentIndex 设置为 -1 将清除突出显示并将currentItem设置为 null。

如果highlightFollowsCurrentItem是true，设置这些属性中的任何一个都会平滑滚动ListView以使当前项变得可见。

请注意，当前项的位置可能只是近似的，直到它在视图中可见。


currentItem : Item

该currentIndex属性保存当前项的索引，并currentItem保存当前项。将currentIndex设置为 -1 将清除突出显示并将 currentItem 设置为 null。

如果highlightFollowsCurrentItem是true，设置这些属性中的任何一个都会平滑滚动ListView以使当前项变得可见。

请注意，当前项的位置可能只是近似的，直到它在视图中可见。


currentSection : string

该属性保存当前位于视图开头的部分。


delegate : Component

委托提供了一个模板，定义了视图实例化的每个项。索引作为可访问index属性公开。根据数据模型的类型，模型的属性也可用。

委托中的对象和绑定的数量对视图的闪烁性能有直接影响。如果可能，将正常显示委托不需要的功能放在Loader中，Loader可以在需要时加载其他组件。

ListView将根据委托中根项的大小对项进行布局。

建议代理的大小为整数，以避免项的亚像素对齐。

委托实例的默认堆叠顺序是1.

注意：委托会根据需要进行实例化，并且可以随时销毁。它们是ListView的contentItem的父级，而不是视图本身。状态永远不应该存储在委托中。

请参考ListView 中的堆叠顺序章节。


displaced : Transition

此属性保存通用转换以应用于已被影响视图的任何模型操作置换的项。

这可以方便地指定要应用于由添加、移动或删除操作替换的任何项的通用转换，而无需指定单独的addDisplaced、moveDisplaced和removeDisplaced属性。例如，这是一个指定置换转换的视图：


  ListView {
      ...
      displaced: Transition {
          NumberAnimation { properties: "x,y"; duration: 1000 }
      }
  }

当在上面的视图中添加、移动或删除任何项时，它下面的项会被移位，导致它们在视图中向下移动（或横向移动，如果水平方向）。当发生这种位移时，项在视图中向其新 x,y 位置的移动将由NumberAnimation在一秒钟内按照指定的方式进行动画处理。

如果视图指定了此通用置换过渡以及特定的addDisplaced、moveDisplaced或removeDisplaced过渡，则在发生相关操作时，将使用更具体的过渡而不是通用的置换过渡，前提是尚未禁用更具体的过渡（通过将启用设置为假）。如果它确实已被禁用，则改为应用通用置换过渡。

有关如何使用视图转换的更多详细信息和示例，请参阅ViewTransition文档。

请参考 addDisplaced，moveDisplaced，removeDisplaced 和 ViewTransition。


displayMarginBeginning : int

此属性允许代理在视图几何图形之外显示。

如果此值不为零，则视图将在视图开始之前或结束之后创建额外的委托。视图将创建尽可能多的委托，因为它可以适应指定的像素大小。

例如，如果在垂直视图中，代理的高度为 20 像素，displayMarginBeginning并且displayMarginEnd都设置为 40，则将创建并显示上方的 2 个代理和下方的 2 个代理。

默认值为 0。

此属性旨在允许某些 UI 配置，而不是作为性能优化。如果您出于性能原因希望在视图几何之外创建委托，您可能希望使用cacheBuffer属性。

这个属性是在 QtQuick 2.3 中引入的。


displayMarginEnd : int

此属性允许代理在视图几何图形之外显示。

如果此值不为零，则视图将在视图开始之前或结束之后创建额外的委托。视图将创建尽可能多的委托，因为它可以适应指定的像素大小。

例如，如果在垂直视图中，代理的高度为 20 像素，displayMarginBeginning并且displayMarginEnd都设置为 40，则将创建并显示上方的 2 个代理和下方的 2 个代理。

默认值为 0。

此属性旨在允许某些 UI 配置，而不是作为性能优化。如果您出于性能原因希望在视图几何之外创建委托，您可能希望使用cacheBuffer属性。

这个属性是在 QtQuick 2.3 中引入的。


effectiveLayoutDirection : enumeration

该属性保存水平方向列表的有效布局方向。

当为区域布局使用附加属性LayoutMirroring::enabled时，水平列表的视觉布局方向将被镜像。但是，属性layoutDirection将保持不变。

请参考 ListView::layoutDirection 和 LayoutMirroring。


footer : Component

此属性保存要用作页脚的组件。

为每个视图创建一个页脚组件的实例。页脚位于视图的末尾，位于任何项之后。页脚的默认堆叠顺序是1。

请参考 header，footerItem 和 在ListView 中堆叠顺序。


footerItem : Item

这包含从页脚组件创建的页脚项。

为每个视图创建一个页脚组件的实例。页脚位于视图的末尾，位于任何项之后。页脚的默认堆叠顺序是1.

请参考 footer，headerItem 和 在ListView 中堆叠顺序。


footerPositioning : enumeration

此属性确定页脚项的定位。

可能的值是：

  • ListView.InlineFooter （默认） - 页脚位于内容的末尾，并像普通项一样与内容一起移动。
  • ListView.OverlayFooter - 页脚位于视图的末尾。
  • ListView.PullBackFooter - 页脚位于视图的末尾。可以通过向后移动内容来推开页脚，并通过向前移动内容来拉回页脚。

注意：此属性对页脚的堆叠顺序没有影响。例如，如果在使用时，页脚应显示在委托项上方ListView.OverlayFooter，则其 Z 值应设置为高于委托的值。有关详细信息，请参阅ListView 中的堆叠顺序。

这个属性是在 Qt 5.4 中引入的。


header : Component

此属性保存要用作标题的组件。

为每个视图创建一个标题组件的实例。标头位于视图的开头，在任何项之前。页眉的默认堆叠顺序是1.

请参考 footer，headerItem 和 在ListView 中堆叠顺序。


headerItem : Item

这包含从标题组件创建的标题项。

为每个视图创建一个标题组件的实例。标头位于视图的开头，在任何项之前。页眉的默认堆叠顺序是1.

请参考 header，footerItem 和 在ListView 中堆叠顺序。


headerPositioning : enumeration

此属性确定标题项的定位。

可能的值是：

  • ListView.InlineHeader （默认） - 标题位于内容的开头，并像普通项一样与内容一起移动。
  • ListView.OverlayHeader - 标题位于视图的开头。
  • ListView.PullBackHeader - 标题位于视图的开头。标题可以通过向前移动内容来推开，并通过向后移动内容来拉回。

注意：此属性对页眉的堆叠顺序没有影响。例如，使用 时，如果标题应显示在委托项上方ListView.OverlayHeader，则其 Z 值应设置为高于委托的值。有关详细信息，请参阅ListView 中的堆叠顺序。

这个属性是在 Qt 5.4 中引入的。


highlight : Component

此属性保存要用作突出显示的组件。

为每个列表创建一个突出显示组件的实例。生成的组件实例的几何图形由列表管理，以便与当前项保持一致，除非highlightFollowsCurrentItem属性为 false。高亮项的默认堆叠顺序是0.

请参考 highlightItem，highlightFollowsCurrentItem，ListView高亮的例子 和 在ListView 中堆叠顺序。


highlightFollowsCurrentItem : bool

该属性保存高亮是否由视图管理。

如果此属性为 true（默认值），则突出显示平滑移动以跟随当前项。否则，高亮不会被视图移动，任何移动都必须由高亮执行。

这是一个亮点，其运动由SpringAnimation项定义：


  Component {
      id: highlight
      Rectangle {
          width: 180; height: 40
          color: "lightsteelblue"; radius: 5
          y: list.currentItem.y
          Behavior on y {
              SpringAnimation {
                  spring: 3
                  damping: 0.2
              }
          }
      }
  }

  ListView {
      id: list
      width: 180; height: 200
      model: ContactModel {}
      delegate: Text { text: name }

      highlight: highlight
      highlightFollowsCurrentItem: false
      focus: true
  }

请注意，突出显示动画也会影响视图滚动的方式。这是因为视图移动以将高亮保持在首选高亮范围（或可见视口）内。

请参考 highlight 和 highlightMoveVelocity。


highlightItem : Item

这包含从突出显示组件创建的突出显示项。

highlightItem由视图管理，除非highlightFollowsCurrentItem被设置为false。突出显示项的默认堆叠顺序为0。

请参考 highlight，highlightFollowsCurrentItem 和 在ListView 中堆叠顺序。


highlightMoveDuration : int

这些属性控制突出显示代理的移动速度和调整动画大小。

highlightFollowsCurrentItem必须为 true 才能使这些属性生效。

速度属性的默认值为 400 像素/秒。持续时间属性的默认值为 -1，即高光将花费尽可能多的时间以设定的速度移动。

这些属性与SmoothedAnimation具有相同的特征:如果速度和持续时间都被设置了，动画将使用哪个给出较短的持续时间。

移动速度和持续时间属性用于控制由于索引变化引起的移动；例如，当调用incrementCurrentIndex()时。当用户轻弹ListView时，轻弹的速度用于控制移动。

要仅设置一个属性，可以将另一个设置为-1。例如，如果您只想为持续时间而不是速度设置动画，请使用以下代码：


  highlightMoveDuration: 1000
  highlightMoveVelocity: -1

请参考 highlightFollowsCurrentItem。

section.property : string

这些属性确定要评估的表达式和部分标签的外观。

section.property包含作为每个部分基础的属性的名称。

section.criteria持有基于 形成每个部分的标准section.property。该值可以是以下之一：

  • ViewSection.FullString (default) - 根据section.property值创建部分。

确定部分边界时使用不区分大小写的比较。

  • ViewSection.CurrentLabelAtStart - 当前部分标签在移动时粘在视图的开头。
  • ViewSection.NextLabelAtEnd -  下一个部分标签（在所有可见部分之外）在视图移动时粘在视图的末尾。

    注意：启用ViewSection.NextLabelAtEnd需要视图提前扫描下一部分，这会影响性能，尤其是对于较慢的模型。

列表中的每个项都有名为ListView.section和ListView.previousSection的附加属性ListView.nextSection。

例如，这里是一个ListView，它显示了一个动物列表，分成几个部分。ListView中的每个项根据模型项的“大小”属性放置在不同的部分中。委托组件提供了标记每个部分开始的sectionHeading浅蓝色条。


      // The delegate for each section header

              height: childrenRect.height
              color: "lightsteelblue"

              Text {
                  text: section
                  font.bold: true
                  font.pixelSize: 20
              }
          }

      ListView {
          id: view
          anchors.top: parent.top
          anchors.bottom: buttonBar.top
          width: parent.width
          model: animalsModel

返回内容坐标中包含点x、y的可见项。如果在指定的点没有项，或者项不可见，则返回 null。

如果项在可见区域之外，则返回 null，而不管项在滚动到视图时是否存在。

注意：方法只能在组件完成后调用。


positionViewAtIndex(int index, PositionMode mode)

定位视图，使index位于mode指定的位置：

  • ListView.Beginning - 将项定位在视图的顶部（或水平方向的左侧）。
  • ListView.Center - 在视图中心定位项。
  • ListView.End - 将项定位在视图的底部（或水平方向的右侧）。
  • ListView.Visible -  如果项的任何部分可见，则不采取任何行动，否则将项带入视野。
  • ListView.Contain - 确保整个项可见。如果项大于视图，则项位于视图的顶部（或水平方向的左侧）。
  • ListView.SnapPosition -  将项定位在preferredHighlightBegin处。此模式仅在highlightRangeMode为 StrictlyEnforceRange 或通过snapMode启用捕捉时有效。

如果将视图定位在index 处会导致在视图的开头或结尾显示空白空间，则视图将定位在边界处。

不建议使用contentX或contentY将视图定位在特定索引处。这是不可靠的，因为从列表的开头删除项不会导致所有其他项重新定位，并且视图的实际开始可能会根据代表的大小而变化。将项带入视野的正确方法是使用positionViewAtIndex.

注意：方法只能在组件完成后调用。要在启动时定位视图，应由 Component.onCompleted 调用此方法。例如，要将视图定位在末尾：


  Component.onCompleted: positionViewAtIndex(count - 1, ListView.Beginning)
