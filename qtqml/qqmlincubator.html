详细信息

QQmlIncubator类允许QML对象以异步方式创建。

创建 QML 对象（例如视图中的委托或应用程序中的新页面）可能会花费大量时间，尤其是在资源受限的移动设备上。当应用程序直接使用QQmlComponent::create () 时，会同步创建 QML 对象实例，这取决于对象的复杂程度，可能会导致应用程序出现明显的停顿或卡顿。

使用QQmlIncubator 可以更好地控制 QML 对象的创建，包括允许使用应用程序空闲时间异步创建它。以下示例展示了QQmlIncubator的简单使用。


  QQmlIncubator incubator;
  component->create(incubator);

  while (!incubator.isReady()) {
      QCoreApplication::processEvents(QEventLoop::AllEvents, 50);
  }

  QObject *object = incubator.object();

异步孵化器由设置在QQmlEngine上的QQmlIncubationController控制，它让引擎知道应用程序何时处于空闲状态，应该处理孵化对象。如果没有在QQmlEngine上设置孵化控制器，那么QQmlIncubator将同步创建对象，而不考虑指定的IncubationMode。

QQmlIncubator支持三种孵化模式：

  • Synchronous 创建是同步发生的。也就是说，一旦QQmlComponent::create()调用返回，孵化器将已经处于Error或Ready状态。与直接在QQmlComponent上使用同步创建方法相比，同步孵化器并没有真正的优势，但是它可以简化应用程序的实现，为同步和异步创建使用相同的API。
  • Asynchronous (默认) 创建是异步进行的，假设在QQmlEngine上设置了QQmlIncubatorController。

    孵化器将保持在Loading状态，直到创建完成或发生错误。statusChanged()回调可以用来通知状态的变化。

    应用程序应该使用异步孵化模式来创建不立即需要的对象。例如，ListView类型使用异步孵化来创建在列表滚动时稍微远离屏幕的对象。如果在异步创建过程中，立即需要对象，那么可以调用QQmlIncubator::forceCompletion()方法来同步完成创建过程。

  • AsynchronousIfNested 如果是嵌套异步创建的一部分，则创建将异步进行;如果不是，则同步进行。

    在 QML 组件希望出现同步实例化的大多数场景中，它应该使用这种模式。

    最好通过示例来解释此模式。当ListView中首次创建类型，它需要一组初始代表展示的填充本身。如果ListView高 400 像素，并且每个委托高 100 像素，则需要创建四个初始委托实例。如果ListView使用异步孵化模式，则ListView将始终创建为空，然后稍后会出现四个初始项。

    相反，如果ListView使用同步孵化模式，它会正确运行，但可能会导致应用程序卡顿。由于 QML 必须同步停止和实例化ListView的委托，如果ListView是异步实例化的 QML 组件的一部分，这将抵消异步实例化的大部分好处。

    该AsynchronousIfNested模式调和这个问题。通过使用AsynchronousIfNested，如果ListView本身已经是异步实例化的一部分，则ListView委托被异步实例化，否则同步实例化。在嵌套异步实例化的情况下，直到所有嵌套实例化也完成后，外部异步实例化才会完成。这确保在外部异步实例化完成时，像ListView这样的内部项目已经完成加载它们的初始委托。

    使用同步孵化模式几乎总是不正确的 - 想要同步实例化的元素或组件，但没有在应用程序中引入冻结或迟钝的缺点，应该使用AsynchronousIfNested孵化模式。

成员类型文档

enum QQmlIncubator::IncubationMode

指定孵化器运行的模式。不管孵化模式如何，如果QQmlEngine没有设置QQmlIncubationController，那么QQmlIncubator将同步工作。

常量值 描述
QQmlIncubator::Asynchronous0该对象将被异步创建。
QQmlIncubator::AsynchronousIfNested1如果对象是在一个已经是异步创建一部分的上下文中创建的，这个孵化器将加入现有的孵化并异步执行。现有的孵化在它和这个孵化完成之前不会变成就绪。否则，孵化将同步执行。
QQmlIncubator::Synchronous2该对象将被同步创建。

enum QQmlIncubator::Status

Specifies the status of the QQmlIncubator.

常量值 描述
QQmlIncubator::Null0孵化没有进行。调用QQmlComponent::create ()开始孵化。
QQmlIncubator::Ready1对象是完全创建的，可以通过调用object ()来访问。
QQmlIncubator::Loading2对象正在创建过程中。
QQmlIncubator::Error3发生错误。错误可以通过调用errors ()来访问。

成员函数文档

QQmlIncubator::QQmlIncubator(QQmlIncubator::IncubationMode mode = Asynchronous)

新建一个指定mode （模式）的孵化器。

void QQmlIncubator::clear()

清除孵化器。任何进行中的孵化都将被中止。如果孵化器处于就绪状态，则不会删除创建的对象。

QList<QQmlError> QQmlIncubator::errors() const

返回孵化对象时遇到的错误列表。

void QQmlIncubator::forceCompletion()

强制任何正在进行的孵化同步完成。一旦此调用返回，孵化器将不会处于 Loading 状态。

QQmlIncubator::IncubationMode QQmlIncubator::incubationMode() const

返回传递给QQmlIncubator构造函数的孵化模式。

bool QQmlIncubator::isError() const

如果孵化器的status() 为错误，则返回 true 。

bool QQmlIncubator::isLoading() const

如果孵化器的status() 为 Loading，则返回 true 。

bool QQmlIncubator::isNull() const

如果孵化器的status() 为 Null，则返回 true 。

bool QQmlIncubator::isReady() const

如果孵化器的status() 为就绪，则返回 true 。

QObject *QQmlIncubator::object() const

如果状态为就绪，则返回孵化对象，否则返回 0。

[virtual protected] void QQmlIncubator::setInitialState(QObject *object)

在首次创建对象之后调用，但在评估属性绑定之前调用，并且(如果适用的话)会调用QQmlParserStatus::componentComplete()。这相当于QQmlComponent::beginCreate()和QQmlComponent::completeCreate()之间的点，可以用来为对象的属性赋值。

默认实现什么都不做。

QQmlIncubator::Status QQmlIncubator::status() const

返回孵化器的当前状态。

[virtual protected] void QQmlIncubator::statusChanged(QQmlIncubator::Status status)

当孵化器的状态改变时调用。status是新的状态。

默认实现什么都不做。
