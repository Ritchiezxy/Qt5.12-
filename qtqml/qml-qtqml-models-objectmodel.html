QML中的ObjectModel类型

详细信息

ObjectModel包含要在视图中使用的可视项。当在视图中使用ObjectModel时，视图不需要委托，因为ObjectModel已经包含了可视委托(项)。

项目可以通过index 附加属性确定其在模型中的索引。

下面的示例在ListView中放置了三个彩色矩形。


  import QtQuick 2.0
  import QtQml.Models 2.1

  Rectangle {
      ObjectModel {
          id: itemModel
          Rectangle { height: 30; width: 80; color: "red" }
          Rectangle { height: 30; width: 80; color: "green" }
          Rectangle { height: 30; width: 80; color: "blue" }
      }

      ListView {
          anchors.fill: parent
          model: itemModel
      }
  }

参见Qt Quick例子 - 视图。

属性文档

count : int

模型中的项数。此属性是只读的。


附加属性文档

ObjectModel.index : int

此附加属性保存模型中此委托项的索引。

它附加到委托的每个实例。


方法文档

append(object item)

在模型的末尾追加一个新项。


  objectModel.append(objectComponent.createObject())

该方法在Qt 5.6中引入。

请参见insert() 和 remove()。


clear()

从模型中清除所有项。

该方法在Qt 5.6中引入。

参见append() 和 remove()。


object get(int index)

返回模型中的以 index为索引项。这允许从JavaScript访问或修改项: 


  Component.onCompleted: {
      objectModel.append(objectComponent.createObject())
      console.log(objectModel.get(0).objectName);
      objectModel.get(0).objectName = "first";
  }

index必须是列表中的一个元素。

该方法在Qt 5.6中引入。

参见append()。


insert(int index, object item)

在位置index处向模型插入一个新项。


  objectModel.insert(2, objectComponent.createObject())

index必须指向列表中现有的项，或列表末尾之后的项(相当于append)。

该方法在Qt 5.6中引入。

参见append() 和 remove()。


move(int from, int to, int n = 1)

将n 个项目from （从）一个位置移动to （到）另一个位置。

from和to范围必须存在;例如，将前3项移动到模型的末尾:


  objectModel.move(0, objectModel.count - 3, 3)

该方法在Qt 5.6中引入。

参见append()。


remove(int index, int n = 1)

从模型中删除index项。

该方法在Qt 5.6中引入。

参见clear()。
