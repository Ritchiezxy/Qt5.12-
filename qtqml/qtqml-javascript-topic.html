集成QML和JavaScript

QML语言使用类似json的语法，并允许将各种表达式和方法定义为JavaScript函数。
它还允许用户导入JavaScript文件并使用这些导入提供的功能。

这允许开发人员和设计人员利用他们所掌握的JavaScript知识来快速开发用户界面和应用程序逻辑。

JavaScript 表达式

QML支持高度集成JavaScript，并允许在JavaScript中定义信号处理程序和方法。QML的另一个核心特性是使用property 属性绑定指定和加强对象属性之间的关系的能力，属性绑定也是使用JavaScript定义的。

有关在QML中使用JavaScript表达式的更多信息，请参阅文档页标题为“QML文档中的JavaScript表达式”。

JavaScript 资源

在JavaScript函数中定义的应用程序逻辑可以被分离到单独的JavaScript文件中，这些文件称为JavaScript资源。有几种不同类型的JavaScript资源，它们具有不同的语义。

有关为QML定义JavaScript资源的更多信息，请参阅文档中标题为Defini在QML中定义JavaScript资源的页面。

JavaScript 导入

QML文档可以导入JavaScript资源，而JavaScript资源可以导入其他JavaScript资源以及QML模块。
这允许应用程序开发人员以模块化的、自包含的文件提供应用程序逻辑。

See the documentation page titled  for more information on how to import JavaScript resources and how to use the functionality they provide.有关如何导入JavaScript资源以及如何使用它们提供的功能的更多信息，请参阅文档页面导入JavaScript资源。

JavaScript主机环境

QML引擎提供的JavaScript环境与web浏览器提供的JavaScript环境有一些不同。
某些限制适用于在环境中运行的代码，并且QML引擎在根上下文中提供了JavaScript开发人员可能不熟悉的各种对象。

These limitations and extensions are documented in the description of the  provided by the QML engine.这些限制和扩展记录在由QML引擎提供的JavaScript主机环境描述中。
