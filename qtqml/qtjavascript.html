编写脚本的应用程序

Qt provides support for application scripting with JavaScript. The following guides and references cover aspects of programming with JavaScript and Qt.

脚本类

以下类为Qt应用程序添加了脚本编写功能。

QJSEngine

评估JavaScript代码的环境

QJSValue

作为Qt/JavaScript数据类型的容器

QJSValueIterator

QJSValue的java风格迭代器

基本用法

要计算脚本代码，您需要创建一个QJSEngine并调用它的evaluate()函数，将脚本代码(文本)作为参数传递。


  QJSEngine engine;
  qDebug() << "the magic number is:" << engine.evaluate("1 + 2").toNumber();

返回值将是计算的结果(表示为QJSValue对象);这可以转换为标准的C++和Qt类型。

通过向脚本引擎注册自定义属性，脚本可以使用自定义属性。通过设置脚本引擎的全局对象的属性，这是最容易做到的:


  engine.globalObject().setProperty("foo", 123);
  qDebug() << "foo times two is:" << engine.evaluate("foo * 2").toNumber();

这将属性放置在脚本环境中，从而使它们对脚本代码可用。

使QObject对脚本引擎可用

任何基于QObject的实例都可以与脚本一起使用。

当QObject被传递给QJSEngine::newQObject()函数时，将创建一个Qt Script包装器对象，该对象可用于使QObject的信号、槽、属性和子对象可供脚本使用。

下面是一个例子，让QObject子类的一个实例可用到名为“myObject”的脚本代码中:


  QJSEngine engine;
  QObject *someObject = new MyObject;
  QJSValue objectValue = engine.newQObject(someObject);
  engine.globalObject().setProperty("myObject", objectValue);

这将在脚本环境中创建一个名为myObject的全局变量。变量充当底层C++对象的代理。注意，脚本变量的名称可以是任何名称;例如，它不依赖于QObject::objectName()。

对应用程序安全的影响

使用JavaScript编写应用程序脚本的安全模型遵循与C++代码相同的模型:用户安装脚本运行他们信任的脚本，就像安装Qt应用程序一样。

为了保持用户的信任，应用程序开发人员不应该评估任意的JavaScript代码。JavaScript引擎的沙箱只是一个语义障碍。该脚本在与应用程序的其他部分相同的进程中执行，具有相同的特权，并共享相同的内存。因此，不需要额外的安全保护就可以访问公开给脚本的C++对象。
