详细信息

Locale 对象只能通过Qt.locale()函数创建。不能直接创建。

Qt.locale()函数返回一个JS Locale对象，该对象表示带有指定名称的区域，其格式为"language[_territory][.codeset][@modifier]" 或 "C"。

语言环境支持默认语言环境的概念，该概念由应用程序启动时系统的语言环境设置确定。如果没有参数传递给 Qt.locale() 则返回默认的语言环境对象。

Locale 对象提供了许多为指定的语言环境提供数据的函数和属性。

Locale 对象也可以传递给Date和Number toLocaleString() 和 fromLocaleString() 方法，以便使用指定的语言环境在字符串之间进行转换。

此示例显示为德语区域设置格式化的当前日期：


  import QtQuick 2.0

  Text {
      text: "The date is: " + Date().toLocaleString(Qt.locale("de_DE"))
  }

以下示例以默认语言环境的正确格式显示指定的数字：


  import QtQuick 2.0

  Text {
      text: "The value is: " + Number(23443.34).toLocaleString(Qt.locale())
  }

Qt Quick Locale的数据基于Common Locale data Repository v1.8.1。

语言环境字符串格式类型

所述MONTHNAME（） ，standaloneMonthName（） ，DAYNAME（）和standaloneDayName（）可以使用下列枚举值，以指定字符串表示的格式设置为一个日期的对象。

  • Locale.LongFormat长版本的日和月名称;例如，返回“January”作为月份名称。
  • Locale.ShortFormat 日和月名称的短版本;例如，返回“Jan”作为月份名称。
  • Locale.NarrowFormat 当空间有限时使用的特殊版本的日和月名称;例如，返回“J”作为一个月的名称。请注意，窄格式可能包含不同月份和天数的相同文本，或者如果地区不支持窄名称，它甚至可能是一个空字符串，因此应该避免将其用于日期格式化。此外，对于系统区域设置，此格式与ShortFormat相同。

属性的文档

amText : string

使用12小时时钟约定指定的时间的“AM”后缀的本地化名称。


decimalPoint : string

保存该区域的小数点字符。


exponential : string

保存此语言环境的指数字符。


firstDayOfWeek : enumeration

根据当前语言环境保存一周的第一天。

  • Locale.Sunday = 0
  • Locale.Monday = 1
  • Locale.Tuesday = 2
  • Locale.Wednesday = 3
  • Locale.Thursday = 4
  • Locale.Friday = 5
  • Locale.Saturday = 6

注意：这些值与 JS 日期 API 匹配，这与Qt::Sunday = 7的 Qt C++ API 不同。


groupSeparator : string

保存此语言环境的组分隔符。


measurementSystem : enumeration

此属性定义用于测量的单位。

  • Locale.MetricSystem 此值表示公制单位，例如米、厘米和毫米。
  • Locale.ImperialUSSystem 此值指示英制单位，例如在美国使用的英寸和英里。s.
  • Locale.ImperialUKSystem 此值指示英制单位，例如在英国使用的英寸和英里。
  • Locale.ImperialSystem 提供兼容性。与 Locale.ImperialUSSystem 相同。

name : string

将此区域设置的语言和国家/地区保存为“language_country”形式的字符串，其中语言是小写的两个字母的 ISO 639 语言代码，国家/地区是大写的两个或三个字母的 ISO 3166 国家/地区代码。


nativeCountryName : string

为语言环境保留国家/地区的本地名称。例如，西班牙语/西班牙语言环境的“España”。

请参考 nativeLanguageName。


nativeLanguageName : string

保存本地语言的本地名称。例如，瑞士-德国语言环境的“Schwiizertüütsch”。

请参考 nativeCountryName。


negativeSign : string

保存此语言环境的负号字符。


percent : string

保存此语言环境的百分比字符。


pmText : string

使用 12 小时制约定指定时间的“PM”后缀的本地化名称。


positiveSign : string

保存此语言环境的正号字符。


textDirection : enumeration

保存语言的文本方向：

  • Qt.LeftToRight
  • Qt.RightToLeft

uiLanguages : Array<string>

按优先顺序返回用于翻译目的的区域设置名称的有序列表。

返回值表示用户希望在其中看到 UI 翻译的区域设置名称。

列表中的第一项是最受欢迎的。


weekDays : Array<int>

保存根据当前区域设置被视为工作日的天数数组，其中星期日为 0，星期六为 6。

请参考 firstDayOfWeek。


zeroDigit : string

Holds 返回此语言环境的零位字符。


Method Documentation

string currencySymbol(format)

返回指定format（格式）的货币符号：

  • Locale.CurrencyIsoCode 货币的 ISO-4217 代码。
  • Locale.CurrencySymbol 货币符号。
  • Locale.CurrencyDisplayName 用户可读的货币名称。

请参考 Number::toLocaleCurrencyString()。


string dateFormat(type)

返回用于当前语言环境的日期格式。type指定要返回的FormatType。

请参考 Date。


string dateTimeFormat(type)

返回用于当前语言环境的日期时间格式。type指定要返回的FormatType。

请参考 Date。


string dayName(day, type)

以type指定的可选FormatType返回day的本地化名称（其中 0 代表星期日，1 代表星期一等等）。

请参考 monthName() 和 standaloneDayName()。


string monthName(month, type)

以type指定的可选FormatType返回月份(0-11)的本地化名称。

注意： 所述QLocale C ++ API期望的范围内的（1-12），然而Locale.monthName（）预计为0-11％的JS Date对象。

请参考 dayName() 和 standaloneMonthName()。


string standaloneDayName(day, type)

返回的本地化名称day （其中0代表星期日，1代表星期一等），其作为一个独立的文本，在FormatType由指定的type。

如果区域设置信息未指定独立日期名称，则返回值与dayName() 中的相同。

请参考 dayName() 和 standaloneMonthName()。


string standaloneMonthName(month, type)

以type指定的可选FormatType返回用作独立文本的month(0-11)的本地化名称。

如果语言环境信息未指定独立月份名称，则返回值与monthName() 中的相同。

注意： 所述QLocale C ++ API期望的范围内的（1-12），然而Locale.standaloneMonthName（）预计为0-11％的JS Date对象。

请参考 monthName() 和 standaloneDayName()。


string timeFormat(type)

返回用于当前语言环境的时间格式。type指定要返回的FormatType。

请参考 Date。
