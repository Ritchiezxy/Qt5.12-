详细描述

用于将QFileSelector应用于QML文件加载的类。

QQmlFileSelector将自动将QFileSelector 应用于qml文件和资产路径。

使用方法如下:


  QQmlEngine engine;
  QQmlFileSelector* selector = new QQmlFileSelector(&engine);

然后你可以像这样交换文件:


  main.qml
  Component.qml
  asset.png
  +unix/Component.qml
  +mac/asset.png

在本例中，main.qml通常将Component.qml用于组件类型。但是在unix平台上，将使用unix选择器，并使用+unix/Component.qml版本。注意，这类似于用+unix/Component.qml替换Component.qml，因此在使用Component.qml时，不需要根据选择的版本更改任何路径。

例如，要传递"asset.png"文件路径，您可以像在所有main.qml、Component.qml和+linux/Component.qml中引用"asset.png"一样引用它。在所有情况下，它将被替换为mac平台上的+mac/asset.png。

有关可用选择器的列表，请参见QFileSelector。

您的平台还可能提供额外的选择器供您使用。正如QFileSelector所指定的，用于选择的目录必须以“+”字符开头，因此除非您的项目中有具有这样名称的目录，否则不会意外触发此功能。

如果在引擎上设置了新的QQmlFileSelector，则将替换旧的。使用QQmlFileSelector::get() 查询或使用现有实例。

成员函数文档

QQmlFileSelector::QQmlFileSelector(QQmlEngine *engine, QObject *parent = nullptr)

使用父对象parent创建一个新的QQmlFileSelector，其中包含自己的QFileSelector. engine，它是您希望应用文件选择器的QQmlEngine。它还将拥有QQmlFileSelector的所有权。

[override virtual] QQmlFileSelector::~QQmlFileSelector()

销毁QQmlFileSelector对象。

[static] QQmlFileSelector *QQmlFileSelector::get(QQmlEngine *engine)

获取目标engine上当前活动的QQmlFileSelector。

QFileSelector *QQmlFileSelector::selector() const

返回QFileSelector使用的QQmlFileSelector实例。

这个函数是在Qt 5.7中引入的。

参见setSelector()。

void QQmlFileSelector::setExtraSelectors(QStringList &strings)

将strings中包含的额外选择器添加到当前使用的QFileSelector 。当您只需要使用额外的选择器就可以避免创建自己的QFileSelector 实例时，请使用此选项。

void QQmlFileSelector::setExtraSelectors(const QStringList &strings)

将strings中包含的额外选择器添加到当前使用的QFileSelector 。当您只需要使用额外的选择器就可以避免创建自己的QFileSelector 实例时，请使用此选项。

void QQmlFileSelector::setSelector(QFileSelector *selector)

将QFileSelector 实例设置为选择器，以便QQmlFileSelector使用。QQmlFileSelector不拥有新QFileSelector的所有权。要重置QQmlFileSelector以使用其内部QFileSelector实例，请调用setSelector(0)。

参见selector()。
