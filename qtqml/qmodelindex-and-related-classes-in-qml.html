QModelIndex和QML中的相关类

自Qt 5.5以来，QModelIndex和QPersistentModelIndex在QML中作为基于值的类型公开。同样以类似的方式公开的还有QModelIndexList、QItemSelectionRange和QItemSelection。这些类型中的所有对象都可以作为var 属性或普通JavaScript变量在QML和C++之间来回传递

下面是对这些类暴露给QML的API的概述。要了解更多信息，请参考他们的C++文档。

注意：由于所有这些类型都公开为gadget，因此不会发出属性更改通知信号。因此，绑定到它们的属性可能不会得到预期的结果。对于QPersistentModelIndex尤其如此。

QModelIndex和QPersistentModelIndex类型

  • row : int
  • column : int
  • parent : QModelIndex
  • valid : bool
  • model : QAbstractItemModel
  • internalId : quint64

所有这些属性都是只读的，它们在C++中的对应属性也是如此。

注意：通常的警告适用于QML中的QModelIndex。如果底层模型更改或被删除，那么访问其属性可能会变得危险。因此，您不应该存储任何QModelIndex对象。但是，您可以以安全的方式存储QPersistentModelIndexe 对象。

QModelIndexList 类型

QModelIndexList在QML中作为JavaScript数组公开。C++和C++之间会自动进行转换。事实上，任何JavaScript数组都可以转换回QModelIndexList，用无效的QModelIndex替换非QModelIndex对象。

注意： QModelIndex到QPersistentModelIndex的转换发生在访问数组元素时，因为任何QModelIndexList属性在以这种方式公开时都会保留引用语义。

QItemSelectionRange 类

  • top : int
  • left : int
  • bottom : int
  • right : int
  • width : int
  • height : int
  • topLeft : QPersistentModelIndex
  • bottomRight : QPersistentModelIndex
  • parent : QModelIndex
  • valid : bool
  • empty : bool
  • model : QAbstractItemModel

所有这些属性都是只读的，它们在C++中的对应属性也是如此。此外，我们还公开了以下函数:

  • bool contains(QModelIndex index)
  • bool contains(int row, int column, QModelIndex parentIndex)
  • bool intersects(QItemSelectionRange other)
  • QItemSelectionRange intersected(QItemSelectionRange other)

QItemSelection Type

与QModelIndexList类似，QItemSelection在QML中作为一个QItemSelectionRange对象的JavaScript数组公开。C++和C++之间会自动进行转换。事实上，任何JavaScript数组都可以转换回QItemSelection，用空的QItemSelectionRange替换非QItemSelectionRange对象。

请参考 ItemSelectionModel。
