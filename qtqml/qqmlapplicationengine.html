详细信息

QQmlApplicationEngine提供了一种从单个 QML 文件加载应用程序的便捷方法。

此类结合了QQmlEngine和QQmlComponent以提供一种方便的方法来加载单个 QML 文件。它还向 QML 公开了一些核心应用程序功能，C++/QML 混合应用程序通常会从 C++ 控制这些功能。

它可以像这样使用：


  #include <QGuiApplication>
  #include <QQmlApplicationEngine>

  int main(int argc, char *argv[])
  {
      QGuiApplication app(argc, argv);
      QQmlApplicationEngine engine("main.qml");
      return app.exec();
  }

与QQuickView不同，QQmlApplicationEngine不会自动创建根窗口。如果您使用 Qt Quick 中的可视项，则需要将它们放置在Window 内。

如果你不使用任何需要QQmlApplicationEngine的QML模块(如QtQuick)，你也可以使用同时使用QCoreApplication和QQmlApplicationEngine。

来自默认QQmlEngine的配置更改列表：

  • 连接 Qt.quit() 到 QCoreApplication::quit()
  • 从与主 QML 文件相邻的 i18n 目录自动加载翻译文件。
  • 如果场景包含QQuickWindow ，则自动设置孵化控制器。
  • 自动设置QQmlFileSelector作为url拦截器，将文件选择器应用到所有QML文件和资源。

可以使用从QQmlEngine继承的方法进一步调整引擎行为。

成员函数文档

QQmlApplicationEngine::QQmlApplicationEngine(QObject *parent = nullptr)

使用给定的parent创建一个新的QQmlApplicationEngine。您必须稍后调用load () 才能加载 QML 文件。

QQmlApplicationEngine::QQmlApplicationEngine(const QUrl &url, QObject *parent = nullptr)

创建一个新的QQmlApplicationEngine并用给定的url加载 QML 文件。这是为了方便而提供的，与使用空构造函数并在之后调用 load 相同。

QQmlApplicationEngine::QQmlApplicationEngine(const QString &filePath, QObject *parent = nullptr)

创建一个新的QQmlApplicationEngine并在给定的filePath加载 QML 文件，该文件必须是本地文件路径。如果给出了相对路径，那么它将被解释为相对于应用程序的工作目录。

这是为了方便而提供的，与使用空构造函数并在之后调用 load 相同。

[override virtual] QQmlApplicationEngine::~QQmlApplicationEngine()

销毁QQmlApplicationEngine和它加载的所有 QML 对象。

[slot] void QQmlApplicationEngine::load(const QUrl &url)

加载位于url的根 QML 文件。由文件定义的对象树会立即为本地文件 url 创建。远程 url 是异步加载的，侦听objectCreated信号以确定对象树何时准备就绪。

如果发生错误，则发出objectCreated信号，并使用空指针作为参数，并使用qWarning打印错误消息。

[slot] void QQmlApplicationEngine::load(const QString &filePath)

加载位于filePath的根 QML 文件。filePath必须是本地文件的路径。如果filePath是相对路径，则将其视为相对于应用程序的工作目录。文件定义的对象树被立即实例化。

如果发生错误，则会使用qWarning打印错误消息。

[slot] void QQmlApplicationEngine::loadData(const QByteArray &data, const QUrl &url = QUrl())

加载data 中给出的 QML 。由data 定义的对象树被立即实例化。

如果指定了url，它将用作组件的基本 url。这会影响数据和错误消息中的相对路径。

如果发生错误，则会使用qWarning打印错误消息。

[signal] void QQmlApplicationEngine::objectCreated(QObject *object, const QUrl &url)

当对象完成加载时发出此信号。如果加载成功，则object包含一个指向加载对象的指针，否则该指针为 NULL。

该url 的组件对象从还提供了进来。

注意：如果组件的路径是作为包含相对路径的QString提供的，则url将包含文件的完全解析路径。

QList<QObject *> QQmlApplicationEngine::rootObjects() const

返回由QQmlApplicationEngine实例化的所有根对象的列表。这将只包含通过load () 或便利构造函数加载的对象。

注意： 在 Qt 5.9 之前的版本中，此功能被标记为非const.
