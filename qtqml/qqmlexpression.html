详细信息

QQmlExpression类在QML上下文中计算JavaScript。

例如，给定一个main.qml这样的文件：


  import QtQuick 2.0

  Item {
      width: 200; height: 200
  }

以下代码在上述 QML 的上下文中计算 JavaScript 表达式：


  QQmlEngine *engine = new QQmlEngine;
  QQmlComponent component(engine, QUrl::fromLocalFile("main.qml"));

  QObject *myObject = component.create();
  QQmlExpression *expr = new QQmlExpression(engine->rootContext(), myObject, "width * 2");
  int result = expr->evaluate().toInt();  // result = 400

成员函数文档

QQmlExpression::QQmlExpression()

创建无效的QQmlExpression。

由于表达式没有关联的QQmlContext，这将是一个空表达式对象，其值将始终是无效的QVariant。

QQmlExpression::QQmlExpression(QQmlContext *ctxt, QObject *scope, const QString &expression, QObject *parent = nullptr)

创建一个QQmlExpression对象，它是parent的孩子。

表达式，JavaScript 表达式expression 将在 QQmlContext类型的对象ctxt 中执行。如果指定了scope，则在表达式执行期间范围对象的属性也将在范围内。

QQmlExpression::QQmlExpression(const QQmlScriptString &script, QQmlContext *ctxt = nullptr, QObject *scope = nullptr, QObject *parent = nullptr)

创建一个QQmlExpression对象，它是parent的孩子。

该script提供了要计算的表达式、要计算它的上下文以及要计算它的作用域对象。如果提供，ctxt和scope将覆盖由script提供的上下文和范围对象。

请参考 QQmlScriptString。

[override virtual] QQmlExpression::~QQmlExpression()

销毁QQmlExpression实例。

void QQmlExpression::clearError()

清除任何表达式错误。在此之后调用hasError () 将返回 false。

请参考 hasError() 和 error()。

int QQmlExpression::columnNumber() const

返回此表达式的源文件列号。源位置必须事先通过调用setSourceLocation () 设置。

QQmlContext *QQmlExpression::context() const

返回与此表达式关联的QQmlContext，如果没有关联或QQmlContext已被破坏，则返回0 。

QQmlEngine *QQmlExpression::engine() const

返回与此表达式关联的QQmlEngine，如果没有关联或QQmlEngine已被销毁，则返回0 。

QQmlError QQmlExpression::error() const

返回上次调用evaluate () 的任何错误。如果没有错误，这将返回一个无效的QQmlError实例。

请参考 hasError() 和 clearError()。

QVariant QQmlExpression::evaluate(bool *valueIsUndefined = nullptr)

评估表达式，返回评估结果，如果表达式无效或有错误，则返回无效的QVariant。

valueIsUndefined is set to true if the expression resulted in an undefined value.

请参考 hasError() 和 error()。

QString QQmlExpression::expression() const

返回表达式字符串。

请参考 setExpression()。

bool QQmlExpression::hasError() const

如果最后一次调用evaluate () 导致错误，则返回true ，否则返回false。

请参考 error() 和 clearError()。

int QQmlExpression::lineNumber() const

返回此表达式的源文件行号。源位置必须事先通过调用setSourceLocation () 设置。

bool QQmlExpression::notifyOnValueChanged() const

如果在表达式的计算值更改时发出valueChanged () 信号，则返回 true 。

请参考 setNotifyOnValueChanged().

QObject *QQmlExpression::scopeObject() const

返回表达式的范围对象（如果提供），否则返回 0。

除了表达式的QQmlContext提供的数据外，作用域对象的属性也在表达式求值期间在作用域内。

void QQmlExpression::setExpression(const QString &expression)

将表达式设置为expression。

请参考 expression()。

void QQmlExpression::setNotifyOnValueChanged(bool notifyOnChange)

设置当表达式的计算值更改时是否发出valueChanged () 信号。

如果notifyOnChange为true，QQmlExpression将监视表达式计算中涉及的属性，如果它们发生更改，则发出QQmlExpression::valueChanged ()。这允许应用程序确保与表达式结果关联的任何值保持最新。

如果notifyOnChange为false（默认），QQmlExpression将不会监视表达式计算中涉及的属性，并且永远不会发出QQmlExpression::valueChanged ()。如果应用程序想要对表达式进行“一次性”评估，这会更有效。

请参考 notifyOnValueChanged()。

void QQmlExpression::setSourceLocation(const QString &url, int line, int column = 0)

将此表达式的位置设置为url 的行和列。此信息由脚本引擎使用。

QString QQmlExpression::sourceFile() const

返回此表达式的源文件 URL。源位置必须事先通过调用setSourceLocation () 设置。

[signal] void QQmlExpression::valueChanged()

每次表达式值从上次计算时发生变化时发出。在发出此信号之前，表达式必须至少被评估一次（通过调用QQmlExpression::evaluate ()）。
