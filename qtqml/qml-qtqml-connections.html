 QML的Connections（连接） 类型

描述与信号的一般连接。 更多...

导入语句： import QtQml 2.12
  • 所有成员列表，包括继承成员。

属性

  • enabled : bool
  • ignoreUnknownSignals : bool
  • target : Object

详细信息

一个Connections对象创建到QML信号的连接。

当连接到QML中的信号时，通常的方法是创建一个“on<Signal>”处理程序，它在接收到信号时做出反应，如下所示:


  MouseArea {
      onClicked: { foo(parameters) }
  }

然而，在某些情况下，不可能以几种方式连接到信号，例如:

  • 需要对同一个信号进行多次连接
  • 在信号发送方范围之外创建连接
  • 连接到QML中未定义的目标

当需要其中任何一种时，都可以使用Connections类型。

例如，上面的代码可以修改为使用一个Connections对象，如下所示:


  MouseArea {
      Connections {
          onClicked: foo(parameters)
      }
  }

更一般地，Connections对象可以是信号发送方以外的其他对象的子对象:


  MouseArea {
      id: area
  }
  // ...



  Connections {
      target: area
      onClicked: foo(parameters)
  }

参考 Qt QML。

属性文档

enabled : bool

此属性保存项目是否接受更改事件。

默认情况下，该属性为true。

这个属性是在Qt 5.7中引入的。


ignoreUnknownSignals : bool

通常，连接到一个不存在的信号会产生运行时错误。

如果此属性设置为true，则此类错误将被忽略。
如果您想连接到不同类型的对象，为每个对象处理不同的信号集，那么这是非常有用的。


target : Object

此属性保存发送信号的对象。

I如果未设置此属性，则目标默认为Connection的父节点。

如果设置为null，则不进行连接，并且忽略任何信号处理程序，直到目标不为空为止。
