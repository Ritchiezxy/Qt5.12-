详细信息

DelegateModelGroup类型提供解决的模型数据的手段DelegateModel的委托项，以及排序和过滤这些委托的项。

DelegateModel中初始的可实例化委托项集合由其项组表示，该项组通常直接反映分配给DelegateModel::model的模型的内容。通过将该DelegateModelGroup的名称赋值给DelegateModel::filterOnGroup属性，可以将该集合更改为DelegateModel::group的任何其他成员的内容。

可以使用get()函数访问DelegateModelGroup 中项的数据，该函数返回有关组成员资格和索引以及模型数据的信息。结合move()函数，这可用于实现视图排序，使用remove()过滤视图中的项目，或使用setGroups()和Package委托将项目分类到不同的视图中。不同的组只有在不相交时才能独立排序。在一个组中移动一个项目也会在它所属的所有其他组中移动它。

可以通过使用insert()函数将数据直接插入DelegateModelGroup来补充来自模型的数据。这可用于将模拟项或占位符项引入到视图中，这些项后来在可用时解析为真实模型数据。

委托项也可以使用create()函数直接从DelegateModelGroup实例化，从而可以在没有伴随视图类型的情况下使用DelegateModel或挑选应实例化的特定项，而不管它们当前是否在视图的可见范围内区域。

请参考 QML 动态视图排序教程。

属性的文档

count : int

此属性保存组中的项的数量。


includeByDefault : bool

此属性保存默认情况下是否将新项分配给该组。


name : string

此属性包含组的名称。

模型中的每个组都必须有一个以小写字母开头的唯一名称。


Signal Documentation

changed(array removed, array inserted)

当项已从组中移除或插入到组中时，会发出此信号。

removed （删除）和inserted （插入）的数组中的每个对象都有两个值；插入或删除的第一个项目的索引以及插入或删除的连续项目数的计数。

每个索引都针对先前的更改进行调整，所有删除的项目都位于任何插入的项之前。

对应的处理程序是onChanged。


Method Documentation

create(int index)

create(int index, jsdict data, array groups = undefined)

create(jsdict data, array groups = undefined)

返回对组中index（索引）处的实例化项的引用。

如果提供了data对象，它将被插入到index处，并且将返回引用此新条目的项。可选的group参数标识新条目应属于的组，如果未指定，则这等于调用create()的组。

create 返回的所有项目都添加到persistedItems组中。当未被任何视图引用时，该组中的项保持实例化。


insert(int index, jsdict data, array groups = undefined)

insert(jsdict data, var groups = undefined)

使用与分配给DelegateModel::model的模型中的角色对应的数据值在DelegateModel 的index处创建一个新条目。

如果未提供索引，则数据将附加到模型中。

可选的groups 参数标识新条目应属于的组，如果未指定，则这等于调用了组插入。

稍后可以使用resolve()函数将插入到DelegateModel 中的数据与DelegateModel::model 中的现有条目合并。这可用于创建稍后由实际数据替换的占位符项。


addGroups(int index, int count, stringlist groups)

将从index 开始的count 项添加到groups。


object get(int index)

返回一个 javascript 对象，描述组中index 处的项目。

返回的对象包含与来自附加的DelegateModel 的委托以及该项目的模型相同的信息。它具有以下特性：

  • model 项的模型数据。这与委托中的模型上下文属性相同
  • groups 项所属组的名称列表。可以编写此属性来更改项的成员资格。
  • inItems 该项是否属于该项组。写入此属性将从组中添加或删除该项。
  • itemsIndex 项组中项的索引。
  • in<GroupName> 是否属于动态组groupName。写入此属性将从组中添加或删除该项。
  • <groupName>Index 动态组groupName中项的索引。
  • isUnresolved 项目是否绑定到分配给DelegateModel::model的模型中的索引。如果项目没有绑定到模型，则返回true，如果绑定到模型则返回false。

move(var from, var to, int count)

Moves count at from in a group to a new position.

注意： The DelegateModel acts as a proxy model: it holds the delegates in a different order than the underlying model has them. Any subsequent changes to the underlying model will not undo whatever reordering you have done via this function.

从index开始移除组中数量为count的项。


removeGroups(int index, int count, stringlist groups)

从index开始移除组中数量为count的项。


resolve(int from, int to)

将from处的未解析项绑定到DelegateModel::model中索引为 to 处的项。

未解析项是其数据已插入到DelegateModelGroup而不是从DelegateModel::model索引派生的条目。解析一个项会将目标索引处的项替换为未解析的项。已解析的项将在其绑定索引处反映源模型的数据，并且会在该索引像任何其他项目一样移动时移动。

如果在DelegateModelGroup onChanged() 处理程序中替换了新项，则其插入和替换将作为原子操作传达给视图，从而创建模型内容未更改的外观，或者如果未解析的模型项目与以前未解决的项只是移动了。


setGroups(int index, int count, stringlist groups)

从index开始为groups设置数量为count的项。


setGroups(int index, int count, stringlist groups)

从index开始为groups设置数量为count的项。
