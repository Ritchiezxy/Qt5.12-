详细信息

QQmlPropertyMap类让你可以在QML绑定使用设置键-值对。

QQmlPropertyMap提供了一种向 UI 层公开域数据的便捷方式。以下示例显示了如何在 C++ 中声明数据，然后在 QML 中访问它。

在 C++ 文件中：


  // create our data
  QQmlPropertyMap ownerData;
  ownerData.insert("name", QVariant(QString("John Smith")));
  ownerData.insert("phone", QVariant(QString("555-5555")));

  // expose it to the UI layer
  QQuickView view;
  QQmlContext *ctxt = view.rootContext();
  ctxt->setContextProperty("owner", &ownerData);

  view.setSource(QUrl::fromLocalFile("main.qml"));
  view.show();

然后，在main.qml：


  Text { text: owner.name + " " + owner.phone }

绑定是动态的 - 每当更新键的值时，绑定到该键的任何内容也将更新。

要检测 UI 层中所做的值更改，您可以连接到valueChanged () 信号。但是，请注意，通过调用insert () 或clear ()进行更改时不会发出valueChanged () - 它仅在从 QML 更新值时发出。

注意：无法从map中删除键；添加键后，您只能修改或清除其关联值。

注意：从QQmlPropertyMap派生类时，请使用受保护的双参数构造函数，以确保该类已正确注册到 Qt元对象系统。

成员函数文档

QQmlPropertyMap::QQmlPropertyMap(QObject *parent = nullptr)

构造一个具有父对象parent的可绑定映射。

[protected] QQmlPropertyMap::QQmlPropertyMap(DerivedType *derived, QObject *parent)

构造一个具有父对象parent的可绑定映射。在从QQmlPropertyMap派生的类中使用此构造函数。

派生的类型用于向元对象系统注册属性映射，这是确保派生类的属性可访问所必需的。此类型必须从QQmlPropertyMap派生。

[override virtual] QQmlPropertyMap::~QQmlPropertyMap()

销毁可绑定的map。

void QQmlPropertyMap::clear(const QString &key)

清除与key关联的值（如果有）。

bool QQmlPropertyMap::contains(const QString &key) const

如果map包含key ，则返回 true 。

请参考 size()。

int QQmlPropertyMap::count() const

这是一个重载函数。

与 size()相同。

void QQmlPropertyMap::insert(const QString &key, const QVariant &value)

将与key关联的值设置为value。

如果密钥不存在，则会自动创建。

bool QQmlPropertyMap::isEmpty() const

如果地图不包含键，则返回 true；否则返回假。

请参考 size()。

QStringList QQmlPropertyMap::keys() const

返回键列表。

已清除的键仍将出现在此列表中，即使它们的关联值是无效的 QVariants。

注意：该函数可以通过元对象系统和 QML 调用。见Q_INVOKABLE。

int QQmlPropertyMap::size() const

返回映射中的键数。

请参考 isEmpty() 和 count()。

[virtual protected] QVariant QQmlPropertyMap::updateValue(const QString &key, const QVariant &input)

返回要为键key存储的新值。提供此函数是为了从 QML 拦截对属性的更新，其中 QML 提供的值为input。

重写此函数以在更新时操作属性值。请注意，此函数仅在从 QML 更新值时调用。

QVariant QQmlPropertyMap::value(const QString &key) const

返回与key关联的值。

如果没有为该键设置值（或者如果该值已被清除），则返回一个无效的QVariant。

[signal] void QQmlPropertyMap::valueChanged(const QString &key, const QVariant &value)

每当映射中的一个值发生更改时，就会发出此信号。key是与更改的值对应的键。

注意： valueChanged()在通过调用insert () 或clear ()进行更改时不会发出- 只有在从 QML 更新值时才会发出。

QVariant &QQmlPropertyMap::operator[](const QString &key)

返回与该键相关联的key作为可修改的参考。

如果映射不包含键为 key 的项，则该函数将一个无效的QVariant插入键为key的映射中，并返回对它的引用。

请参考 insert() 和 value()。

QVariant QQmlPropertyMap::operator[](const QString &key) const

这是一个重载函数。

与 value()相同。
