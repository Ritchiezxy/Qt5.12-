详细信息

QJSValue类作为Qt / JavaScript的数据类型的容器。

QJSValue支持ECMA-262标准中定义的类型：原始类型，即 Undefined、Null、Boolean、Number 和 String；以及对象和数组类型。此外，还为 Qt/C++ 类型（例如QVariant和QObject）提供了内置支持。

对于基于对象的类型(包括Date和RegExp)，使用QJSEngine中的newT()函数(例如QJSEngine::newObject())来创建所需类型的QJSValue。对于基元类型，使用其中一个QJSValue构造函数重载。对于其他类型，例如已注册的小工具类型，如QPoint，您可以使用QJSEngine::toScriptValue。

名为 isT() 的方法（例如isBool ()、isUndefined ()）可用于测试值是否属于某种类型。名为 toT() 的方法（例如toBool ()、toString ()）可用于将QJSValue转换为另一种类型。您还可以使用通用的 qjsvalue_cast() 函数。

对象值有零个或多个属性，它们本身就是 QJSValues。使用setProperty ()设置对象的属性，调用property ()获取属性的值。


  QJSEngine myEngine;
  QJSValue myObject = myEngine.newObject();
  QJSValue myOtherObject = myEngine.newObject();
  myObject.setProperty("myChild", myOtherObject);
  myObject.setProperty("name", "John Doe");

如果要遍历脚本对象的属性，请使用QJSValueIterator类。

对象值有一个内部prototype属性，可以通过prototype ()和setPrototype ()访问。

可以通过调用call ()来调用函数对象（isCallable ()为其返回 true 的对象）。构造函数可以通过调用callAsConstructor ()来构造新的对象。

使用equals()或strictlyEquals()来比较一个QJSValue和另一个QJSValue。

注意，QJSValue为其则IsObject（）为真仅承载到一个实际的对象的引用; 复制QJSValue只会复制对象引用，而不是对象本身。如果您想克隆一个对象（即将一个对象的属性复制到另一个对象），您可以借助for-in脚本代码中的语句或C++ 中的QJSValueIterator来实现。

使用数组

要使用QJSValue创建数组，请使用QJSEngine::newArray ()：


  // Assumes that this class was declared in QML.
  QJSValue jsArray = engine->newArray(3);

要设置数组中的单个元素，请使用setProperty(quint32 arrayIndex, const QJSValue &value)重载。例如，用整数填充上面的数组：


  for (int i = 0; i < 3; ++i) {
      jsArray.setProperty(i, QRandomGenerator::global().generate());
  }

要确定数组的长度，请访问该"length"属性。要访问数组元素，请使用属性(quint32 arrayIndex) 重载。以下代码将我们上面创建的数组读回列表：


  QVector<int> integers;
  const int length = jsArray.property("length").toInt();
  for (int i = 0; i < length; ++i) {
      integers.append(jsArray.property(i).toInt());
  }

请参考 QJSEngine and QJSValueIterator.

enum QJSValue::ErrorType

将此枚举用于 JavaScript 语言特定类型的 Error 对象。

当在 C++ 中模拟语言特性需要使用专门的异常类型时，它们可能很有用。此外，它们可能有助于更清楚地传达某些典型条件，而不是抛出通用的 JavaScript 异常。例如，处理网络和资源定位器的代码可能会发现使用 URIError 类型传播与格式错误的定位器相关的错误很有用。

常量值 描述
QJSValue::GenericError1一个通用的 Error 对象，但不是特定的子类型。
QJSValue::RangeError3值与预期的集合或范围不匹配。
QJSValue::ReferenceError4引用了一个不存在的变量。
QJSValue::SyntaxError5遇到不符合语言语法的无效标记或标记序列。
QJSValue::TypeError6操作数或参数与预期的类型不兼容。
QJSValue::URIError7 URI 处理函数使用不正确或提供的 URI 格式不正确。

这个枚举是在 Qt 5.12 中引入或修改的。

QJSValue QJSValue::call(const QJSValueList &args = ...)

将此QJSValue作为函数调用，将args作为参数传递给函数，并使用 globalObject() 作为“this”对象。返回从函数返回的值。

如果此QJSValue不可调用，则 call() 不执行任何操作并返回未定义的QJSValue。

调用call()会导致脚本引擎发生异常；在这种情况下， call() 返回抛出的值（通常是一个Error对象）。可以对返回值调用isError ()来判断是否发生异常。

请参考isCallable ()、callWithInstance ()和callAsConstructor ()。

QJSValue QJSValue::callAsConstructor(const QJSValueList &args = ...)

创建一个新的Object并调用这个QJSValue作为构造函数，使用创建的对象作为“this”对象并传递args作为参数。如果构造函数调用的返回值是一个对象，则返回该对象；否则返回默认构造的对象。

如果此QJSValue不是函数，则 callAsConstructor() 不执行任何操作并返回未定义的QJSValue。

调用该函数会导致脚本引擎发生异常；在这种情况下，将Error返回抛出的值（通常是一个对象）。可以对返回值调用isError ()来判断是否发生异常。

请参考call ()和QJSEngine::newObject ()。

QJSValue QJSValue::callWithInstance(const QJSValue &instance, const QJSValueList &args = ...)

将此QJSValue作为函数调用，在函数调用中使用instance 作为“this”对象，并将args作为参数传递给函数。返回从函数返回的值。

如果此QJSValue不是函数，则call () 不执行任何操作并返回未定义的QJSValue。

请注意，如果instance不是对象，则全局对象（请参阅QJSEngine::globalObject ()）将用作“this”对象。

调用call ()会导致脚本引擎发生异常；在这种情况下，call () 返回抛出的值（通常是一个Error对象）。可以对返回值调用isError ()来判断是否发生异常。

请参考 call()。

bool QJSValue::deleteProperty(const QString &name)

尝试删除此对象的给定name的属性。如果属性被删除，则返回true，否则返回false。

此函数的行为与 JavaScript 的删除操作符一致。特别是：

  • 无法删除不可配置的属性。
  • 即使此对象没有给定名称的属性（即，不存在的属性是“可简单删除的”），此函数也将返回 true 。
  • 如果这个对象没有自己的给定name的属性，但是prototype ()链中的一个对象有，则原型对象的属性不会被删除，这个函数返回true。

请参考setProperty ()和hasOwnProperty ()。

bool QJSValue::equals(const QJSValue &other) const

如果这个QJSValue等于other，则返回true，否则返回false。比较遵循ECMA-262第11.9.3节“抽象等式比较算法”中描述的行为。

即使这个QJSValue的类型与other 的类型不同，这个函数也可以返回true;也就是说，比较不严格。例如，比较数字9和字符串“9”返回true;将未定义的值与空值进行比较返回true;比较原始值为6的Number 对象和原始值为"6"的String 对象返回true;将数字1与布尔值true比较返回true。如果您想在不进行隐式值转换的情况下执行比较，请使用strictlyEquals()。

请注意，如果此QJSValue或other 值是对象，则调用此函数会对脚本引擎产生副作用，因为引擎将调用对象的 valueOf() 函数（可能还有toString ()）以尝试将对象转换为原始值（可能导致未捕获的脚本异常）。

请参考 strictlyEquals()。

QJSValue QJSValue::property(quint32 arrayIndex) const

这是一个重载函数。

返回给定arrayIndex处的属性。

可以通过两种方式访问​​数组中的元素。第一种是使用数组索引作为属性名称：


  qDebug() << jsValueArray.property(QLatin1String("4")).toString();

第二种是使用带索引的重载：


  qDebug() << jsValueArray.property(4).toString();

这两种方法都达到了相同的结果，只是后者：

  • 更容易使用（可以直接使用整数）
  • 更快（不转换为整数）

如果此QJSValue不是 Array 对象，则此函数的行为就像使用arrayIndex的字符串表示调用property ()一样。

void QJSValue::setPrototype(const QJSValue &prototype)

如果这个QJSValue是一个对象，则设置__proto__这个对象的内部原型（属性）为原型；如果QJSValue为空，则将原型设置为空；否则什么都不做。

bool QJSValue::strictlyEquals(const QJSValue &other) const

如果此QJSValue等于other使用严格比较（无转换），则返回 true，否则返回 false。比较遵循ECMA-262第 11.9.6 节“严格相等比较算法”中描述的行为。

如果此QJSValue的类型与other值的类型不同，则此函数返回 false。如果类型相等，则结果取决于类型，如下表所示：

类型 结果
Undefinedtrue
Nulltrue
Boolean如果两个值都为true，则为true，否则为false
Number如果任一值为 NaN（非数字），则为 false；如果值相等则为ture，否则为false
String如果两个值是完全相同的字符序列，则为 true，否则为 false
Object如果两个值都指向同一个对象，则为 true，否则为 false

请参考 equals()。

QJSValue &QJSValue::operator=(const QJSValue &other)

请注意，如果other是一个对象（isObject () 返回 true），则只会分配对底层对象的引用；对象本身不会被复制。
