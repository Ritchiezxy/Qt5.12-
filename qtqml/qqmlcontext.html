详细描述

QQmlContext类定义QML引擎中的上下文。

上下文允许向 QML 引擎实例化的 QML 组件公开数据。

每个QQmlContext 都包含一组与其QObject属性不同的属性，这些属性允许通过名称将数据显式绑定到上下文。上下文属性通过调用QQmlContext::setContextProperty ()来定义和更新。下面的示例显示了一个 Qt 模型被绑定到一个上下文，然后从 QML 文件访问。


  QQmlEngine engine;
  QStringListModel modelData;
  QQmlContext *context = new QQmlContext(engine.rootContext());
  context->setContextProperty("myModel", &modelData);

  QQmlComponent component(&engine);
  component.setData("import QtQuick 2.0\nListView { model: myModel }", QUrl());
  QObject *window = component.create(context);

请注意，删除它构造的任何QQmlContext是创建者的责任。如果在销毁组件实例context时不再需要示例中的对象window，则context必须显式销毁该对象。确保这一点的最简单方法是设置window为context。

为了简化绑定和维护更大的数据集，可以在QQmlContext上设置上下文对象。上下文对象的所有属性都可以在上下文中通过名称使用，就好像它们都是通过调用QQmlContext::setContextProperty ()单独添加的。通过属性的通知信号检测对属性值的更改。设置上下文对象比手动添加和维护上下文属性值更快、更容易。

下面的示例与前一个示例具有相同的效果，但它使用了上下文对象。


  class MyDataSet : ... {
      ...
      Q_PROPERTY(QAbstractItemModel *myModel READ model NOTIFY modelChanged)
      ...
  };

  MyDataSet myDataSet;
  QQmlEngine engine;
  QQmlContext *context = new QQmlContext(engine.rootContext());
  context->setContextObject(&myDataSet);

  QQmlComponent component(&engine);
  component.setData("import QtQuick 2.0\nListView { model: myModel }", QUrl());
  component.create(context);

由QQmlContext::setContextProperty ()显式添加的所有属性优先于上下文对象的属性。

上下文层次结构

上下文形成层次结构。这个层次结构的根是 QML 引擎的根上下文。子上下文继承其父上下文的属性；如果子上下文设置了其父上下文中已存在的上下文属性，则新上下文属性将覆盖父上下文的属性。

以下示例定义了两个上下文 ——context1和context2。第二个上下文用新值覆盖从第一个上下文继承的“b”上下文属性。


  QQmlEngine engine;
  QQmlContext *context1 = new QQmlContext(engine.rootContext());
  QQmlContext *context2 = new QQmlContext(context1);

  context1->setContextProperty("a", 12);
  context1->setContextProperty("b", 12);

  context2->setContextProperty("b", 15);

虽然在上下文中实例化的QML对象严格地不属于该上下文中，但它们的绑定属于该上下文中。如果一个上下文被销毁，未执行的QML对象的属性绑定将停止计算。

警告： 在该上下文中创建对象后设置上下文对象或添加新的上下文属性是一项昂贵的操作（实质上是强制重新评估所有绑定）。因此，只要有可能，您应该在使用上下文创建任何对象之前完成上下文的“设置”。

请参考 将 C++ 类型的属性暴露给 QML。

成员函数文档

QQmlContext::QQmlContext(QQmlEngine *engine, QObject *parent = nullptr)

创建一个新的QQmlContext作为engine根上下文的子级和QObject parent。

QQmlContext::QQmlContext(QQmlContext *parentContext, QObject *parent = nullptr)

使用给定的parentContext和QObject parent创建一个新的QQmlContext。

[override virtual] QQmlContext::~QQmlContext()

销毁QQmlContext。

任何依赖于此上下文的表达式或子上下文都将失效，但不会被销毁（除非它们是QQmlContext对象的父级）。

QUrl QQmlContext::baseUrl() const

返回组件的基本 url，如果没有设置，则返回包含的组件。

请参考 setBaseUrl()。

QObject *QQmlContext::contextObject() const

返回上下文对象，如果没有上下文对象，则返回 0。

请参考 setContextObject()。

QVariant QQmlContext::contextProperty(const QString &name) const

R将此上下文的name属性值作为QVariant 返回。

请参考 setContextProperty()。

QQmlEngine *QQmlContext::engine() const

返回上下文的QQmlEngine，如果上下文没有QQmlEngine或QQmlEngine被破坏，则返回0 。

bool QQmlContext::isValid() const

返回上下文是否有效。

一个上下文要有效，必须有引擎，而且它的contextObject ()，如果有的话，一定不能被删除。

QString QQmlContext::nameForObject(QObject *object) const

返回此上下文中object 的名称，如果object 未在上下文中命名，则返回空字符串。对象由setContextProperty ()命名，或者在 QML 创建上下文的情况下由 ids命名。

如果对象有多个名称，则返回第一个。

QQmlContext *QQmlContext::parentContext() const

返回上下文的父级QQmlContext，如果此上下文没有父级或父级已被销毁，则返回0。

QUrl QQmlContext::resolvedUrl(const QUrl &src)

解析URL src相对于包含组件的URL。

请参考 QQmlEngine::baseUrl() 和 setBaseUrl()。

void QQmlContext::setBaseUrl(const QUrl &baseUrl)

显式设置 url， resolveUrl () 将用于对baseUrl 的相对引用。

调用此函数将覆盖默认使用的包含组件的 url。

请参考 baseUrl() 和 resolvedUrl()。

void QQmlContext::setContextObject(QObject *object)

设置上下文object。

请参考 contextObject()。

void QQmlContext::setContextProperties(const QVector<QQmlContext::PropertyPair> &properties)

在此上下文中设置一批properties 。

一次性设置所有属性可以避免不必要的刷新表达式，因此建议不要为每个单独的属性调用setContextProperty ()。

这个函数在Qt 5.11中引入。

请参考 QQmlContext::setContextProperty()。

void QQmlContext::setContextProperty(const QString &name, QObject *value)

在此上下文中设置name 属性的value 。

QQmlContext并没有取得其所有权的value。

请参考 contextProperty()。

void QQmlContext::setContextProperty(const QString &name, const QVariant &value)

置该value 的名字在这个上下文属性。
