详细信息

状态对象可以有子状态，也可以有到其他状态的转换。状态是声明性状态机框架的一部分。

有子状态的状态

childMode 属性决定如何处理子状态。对于非并行状态组，必须使用initialState属性设置初始状态。子状态是相互排斥的状态，当父状态是转换的目标时，状态机需要知道要进入哪个子状态。

当输入最后一个子状态(FinalState)时，状态发出State::finished()信号。

errorState设置状态的错误状态。错误状态是状态机在试图进入状态时检测到错误时将转换到的状态(例如，因为没有设置初始状态)。

用例


  import QtQuick 2.0
  import QtQml.StateMachine 1.0 as DSM

  Rectangle {
      DSM.StateMachine {
          id: stateMachine
          initialState: state
          running: true
          DSM.State {
              id: state
          }
      }
  }

参见StateMachine 和 FinalState。

属性文档

childMode : enumeration

此状态的子模式

此属性的默认值是QState.ExclusiveStates。

此枚举指定如何处理一个状态的子状态：

  • QState.ExclusiveStates 子状态是互斥的，必须通过设置initialState、属性来设置初始状态。
  • QState.ParallelStates 子状态是平行的。当输入父状态时，它的所有子状态都是并行输入的。

errorState : QAbstractState

该状态的错误状态。


initialState : QAbstractState

此状态(其子状态之一)的初始状态。


信号文档

finished()

当进入此状态的最终子状态时，将发出此信号。

相应的处理程序是onFinished。

参见QAbstractState::active, QAbstractState::entered和 QAbstractState::exited。
