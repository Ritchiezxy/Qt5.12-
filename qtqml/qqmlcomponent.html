详细信息

QQmlComponent类封装了QML组件定义。

组件是可重用、封装的 QML 类型，具有明确定义的接口。

一个QQmlComponent实例可以从QML文件被创建。例如，如果有这样一个main.qml文件：


  import QtQuick 2.0

  Item {
      width: 200
      height: 200
  }

以下代码将这个 QML 文件作为组件加载，使用create ()创建这个组件的实例，然后查询Item的宽度值：


  QQmlEngine *engine = new QQmlEngine;
  QQmlComponent component(engine, QUrl::fromLocalFile("main.qml"));

  QObject *myObject = component.create();
  QQuickItem *item = qobject_cast<QQuickItem*>(myObject);
  int width = item->width();  // width = 200

要在代码中创建QQmlEngine实例不可用的组件实例，可以使用qmlContext () 或qmlEngine ()。例如，在下面的场景中，正在QQuickItem子类中创建子项：


  void MyCppItem::init()
  {
      QQmlEngine *engine = qmlEngine(this);
      // Or:
      // QQmlEngine *engine = qmlContext(this)->engine();
      QQmlComponent component(engine, QUrl::fromLocalFile("MyItem.qml"));
      QQuickItem *childItem = qobject_cast<QQuickItem*>(component.create());
      childItem->setParentItem(this);
  }

请注意，null当在QObject子类的构造函数中调用时，这些函数将返回，因为实例还没有上下文或引擎。

网络组件

如果传递给QQmlComponent的 URL是网络资源，或者 QML 文档引用了网络资源，则QQmlComponent必须先获取网络数据，然后才能创建对象。在这种情况下，QQmlComponent将具有Loading 状态。在调用QQmlComponent::create ()之前，应用程序必须等待组件Ready（就绪）。

以下示例显示如何从网络资源加载 QML 文件。创建QQmlComponent 后，它会测试该组件是否正在加载。如果是，则连接到QQmlComponent::statusChanged () 信号，否则continueLoading()直接调用该方法。请注意，如果组件已被缓存并立即准备就绪，则网络组件的QQmlComponent::isLoading () 可能为 false。


  MyApplication::MyApplication()
  {
      // ...
      component = new QQmlComponent(engine, QUrl("http://www.example.com/main.qml"));
      if (component->isLoading())
          QObject::connect(component, SIGNAL(statusChanged(QQmlComponent::Status)),
                           this, SLOT(continueLoading()));
      else
          continueLoading();
  }

  void MyApplication::continueLoading()
  {
      if (component->isError()) {
          qWarning() << component->errors();
      } else {
          QObject *myObject = component->create();
      }
  }

成员类型文档

enum QQmlComponent::CompilationMode

指定QQmlComponent应立即加载组件还是异步加载组件。

常量值 描述
QQmlComponent::PreferSynchronous0 更喜欢立即加载/编译组件，阻塞线程。这并不总是可能的；例如，远程 URL 将始终异步加载。
QQmlComponent::Asynchronous1 在后台线程中加载/编译组件。

enum QQmlComponent::Status

指定QQmlComponent的加载状态。

常量值 描述
QQmlComponent::Null0这个QQmlComponent没有数据。调用loadUrl () 或setData () 添加 QML 内容。
QQmlComponent::Ready1这个QQmlComponent已经准备好了，可以调用create ()了。
QQmlComponent::Loading2这个QQmlComponent正在加载网络数据。
QQmlComponent::Error3发生了错误。调用errors () 来检索错误列表。

[virtual] QObject *QQmlComponent::beginCreate(QQmlContext *publicContext)

此方法提供对组件实例创建的高级控制。一般来说，程序员应该使用QQmlComponent::create ()来创建对象实例。

从该组件创建一个对象实例。nullptr如果创建失败则返回。publicContext指定在其中创建对象实例的上下文。

当QQmlComponent构建一个实例，它发生在三个步骤：

  1. 创建对象层次结构，并分配常量值。
  2. 首次评估属性绑定。
  3. 如果适用，在对象上调用QQmlParserStatus::componentComplete ()。

QQmlComponent::beginCreate() 与QQmlComponent::create () 的不同之处在于它只执行第 1步，必须调用QQmlComponent::completeCreate () 才能完成第 2 步和第 3 步。

当使用附加属性将信息传递给实例化组件时，此断点有时很有用，因为它允许在属性绑定生效之前配置它们的初始值。

返回的对象实例的所有权转移给调用者。

请参考 completeCreate() 和 QQmlEngine::ObjectOwnership。

[virtual] void QQmlComponent::completeCreate()

此方法提供对组件实例创建的高级控制。一般来说，程序员应该使用QQmlComponent::create ()来创建组件。

该函数完成了从QQmlComponent::beginCreate ()开始的组件创建，之后必须调用。

请参考 beginCreate()。

[virtual] QObject *QQmlComponent::create(QQmlContext *context = nullptr)

从该组件创建一个对象实例。nullptr如果创建失败则返回。context 指定在其中创建对象实例的上下文。

如果context 是nullptr（默认值），它将在引擎的根上下文中创建实例。

返回的对象实例的所有权转移给调用者。

如果从这个组件创建的对象是一个可视项，它必须有一个可视父项，可以通过调用QQuickItem::setParentItem ()来设置。有关更多详细信息，请参阅概念 - Qt Quick 中的可视化父级。

请参考 QQmlEngine::ObjectOwnership。

void QQmlComponent::create(QQmlIncubator &incubator, QQmlContext *context = nullptr, QQmlContext *forContext = nullptr)

使用提供的incubator从此组件创建对象实例。context 指定在其中创建对象实例的上下文。

如果context 为 0（默认值），它将在引擎的根上下文中创建实例。

forContext指定此对象创建所依赖的上下文。如果forContext是异步创建的，并且QQmlIncubator::IncubationMode是QQmlIncubator::AsynchronousIfNested，则该对象也将异步创建。如果forContext为 0（默认值），则context 将用于此决策。

创建的对象及其创建状态可通过incubator 获得。

请参考 QQmlIncubator。
