详细信息

StateMachine基于Statecharts的概念和表示法。StateMachine是声明式状态机框架的一部分。

状态机管理一组状态和这些状态之间的转换;这些状态和转换定义了状态图。一旦构建了状态图，状态机就可以执行它。StateMachine的执行算法基于状态图XML (SCXML)算法。框架概述给出了几个状态图和构建它们的代码。

在启动机器之前，必须设置initialState。初始状态是机器启动时进入的状态。然后可以将running属性设置为true或在状态机中start()。启动信号在进入初始状态时发出。

状态机处理事件并进行转换，直到进入顶级最终状态;然后状态机发出finished()信号。您还可以显式地stop()状态机(也可以将running属性设置为false)。在这种情况下发出停止信号。

示例使用

下面的代码片段显示了当单击按钮时将完成的状态机:


  import QtQuick 2.0
  import QtQml.StateMachine 1.0 as DSM

  Rectangle {
      Button {
          anchors.fill: parent
          id: button
          text: "Finish state"
          DSM.StateMachine {
              id: stateMachine
              initialState: state
              running: true
              DSM.State {
                  id: state
                  DSM.SignalTransition {
                      targetState: finalState
                      signal: button.clicked
                  }
              }
              DSM.FinalState {
                  id: finalState
              }
              onFinished: Qt.quit()
          }
      }
  }

如果遇到错误，机器将查找errorState，如果有可用的errorState，它将进入该状态。进入错误状态后，可以使用error()检索错误的类型。当进入错误状态时，状态图的执行不会停止。如果没有错误状态应用于错误状态，则机器将停止执行，并将一条错误消息打印到控制台。


请参考 QAbstractState, State, SignalTransition, TimeoutTransition, HistoryState, 和 声明式状态机框架。

属性的文档

[read-only] errorString : string

状态机的错误字符串。


globalRestorePolicy : enumeration

此状态机的状态恢复策略。

该属性的默认值是QState.DontRestoreProperties。

该enum指定恢复策略类型。当机器进入设置一个或多个属性的状态时，恢复策略生效。如果将恢复策略设置为QState.RestoreProperties，状态机将在设置新值之前保存属性的原始值。

稍后，当机器进入没有为给定属性设置值的状态时，该属性将自动恢复为初始值。

对于任何给定的属性，只会保存一个初始值。如果状态机已经保存了某个属性的值，那么在成功恢复该属性之前，它不会被覆盖。

  • QState.DontRestoreProperties 状态机不应该保存属性的初始值并在以后恢复它们。
  • QState.RestoreProperties 状态机应该保存属性的初始值，并在稍后恢复它们。

running : bool

此状态机的运行状态。

请参考 start() 和 stop()。


Signal 文档

started()

当状态机进入初始状态(State::initialState)时发出此信号。

相应的处理程序是onStarted。

请参考 running， start()， 和 State::finished。


stopped()

当状态机停止时发出此信号。

相应的处理程序是onStopped。

请参考 running， stop()，和 State::finished。


方法文档

start()

启动该状态机。机器将重置其配置并转换到初始状态。当输入最终的顶级状态(FinalState)时，机器将发出finished()信号。

注意： 如果没有正在运行的事件循环，状态机将不会运行，例如主应用程序事件循环是由QCoreApplication::exec()()或QApplication::exec()()启动的。

请参考 started, State::finished, stop(), State::initialState, and running.


stop()

停止该状态机。状态机将停止处理事件，然后发出停止信号。

请参考 stopped， start()， 和 运行。
