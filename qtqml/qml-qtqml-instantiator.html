详细信息

Instantiator可用于控制对象的动态创建，或从模板动态创建多个对象。

实例化元素将管理它创建的对象。这些对象是实例化器的父对象，如果实例化器的属性发生更改，实例化器也可以删除这些对象。对象也可以通过其他方法动态销毁，实例化器不会重新创建它们，除非实例化器的属性发生更改。

属性文档

active : bool

当active为真，且委托组件就绪时，实例化器将根据模型创建对象。当active为false时，将不创建任何对象，并且之前创建的任何对象都将被销毁。

默认是正确的。


asynchronous : bool

当异步为真时，实例化器将尝试异步创建对象。这意味着对象可能不会立即可用，即使active被设置为true。

您可以使用objectAdded 信号来响应正在创建的项。

默认是假的false。


count : int

实例化程序当前管理的对象数量。


[default] delegate : QtQml::Component

用于创建所有对象的组件。

请注意，委托的实例中将提供一个额外的变量index。这个变量引用实例化器内部实例的索引，并可用于通过实例化器的objectAt方法获取对象。

如果更改此属性，将销毁所有使用旧委托的实例，并使用新委托创建新实例。


model : variant

此属性可设置为任何受支持的数据模型:

  • 一个数字，表示中继器要创建的委托的数量
  • 模型(例如ListModel项，或QAbstractItemModel子类)
  • 一个字符串列表
  • 一个对象列表

模型的类型影响公开给delegate的属性。

默认值为1，它创建一个委托实例。

参见数据模型。


object : QtObject

这是对第一个创建对象的引用，用于方便只创建了一个对象的情况。


信号文档

objectAdded(int index, QtObject object)

当将对象添加到实例化器时，将发出此信号。index参数保存对象已给定的索引，object参数保存已添加的QtObject。

对应的处理程序是onObjectAdded。


objectRemoved(int index, QtObject object)

当从实例化器中删除对象时，将发出此信号。index参数保存对象已被给定的索引， object参数保存已被删除的QtObject。

如果object是由这个实例化器创建的，则不要保留对该对象的引用，因为在这些情况下，在处理信号后不久就会删除该对象。

对应的处理程序是onObjectRemoved。


方法文档

QtObject objectAt(int index)

返回对具有给定index的对象的引用。
